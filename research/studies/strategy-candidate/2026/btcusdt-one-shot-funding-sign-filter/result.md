# 结论：已结算 funding 符号不足以修复当前单次突破策略

结论为 `DOES_NOT_SUPPORT`。在正式默认 `20×15m Donchian / 2×1m 确认 / 0.5 ATR 最大追价 / 1.5 ATR 止损 / 50%@1.5R / 余量@3R / 最长 1h` 全部固定时，开发期预注册的 LONG、SHORT 两个方向非不利 funding 候选都没有在 favorable、base 或 stress 成本下取得正平均净收益。开发门通过数为 0，按规则停止；2024–2025 evaluation 与 2026H1 confirmation 均未读取。

## 参数空间收敛

1. 关闭“仅用最新已结算 funding 正负号、零阈值过滤当前固定 Donchian/ATR 入场”的分支。
2. 不根据本结果追加 funding 幅度、连续次数、均值、z-score、OI、basis 或波动阈值；这些会把预注册二元问题变成结果驱动调参。
3. 与已失败的 72 组入场近邻及 2/4/8 小时持有期一起看，继续在当前信号族上做联合大网格缺少可吸收成本的毛边际。
4. 当前产品策略、计划、资金和 Demo 状态保持不变；单笔 Demo 盈利不能推翻跨年度成本后反证。

未过滤 LONG/SHORT 基准已与父研究固定默认行逐字段完全一致。研究仍受 1m OHLC、保守同 bar 顺序、成本代理、订单簿/部分成交/延迟/保证金缺失等限制；底层 2021–2026H1 市场区间此前已在其他问题中暴露，因此本结果只能关闭这个具体规则分支，不能外推为“funding 永远无用”或“所有 crypto 趋势策略无效”。
