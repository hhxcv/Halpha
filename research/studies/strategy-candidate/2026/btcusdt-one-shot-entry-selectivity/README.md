# BTCUSDT 单次突破入场选择性研究

## 状态与边界

- 主要类型：`STRATEGY_CANDIDATE`
- 候选标识：`RESEARCH_BTCUSDT_ONE_SHOT_ENTRY_SELECTIVITY_2026`
- 开题产品基准：Git `bbd2458204389f05b58594a9a255b66062080fe3`
- 正式策略背景：`ONE_SHOT_DONCHIAN_ATR_BREAKOUT 1.0.1`、`BTCUSDT-PERP`
- 研究数据截止：`2026-07-01T00:00:00Z`
- 产品影响：无。本问题只筛查候选参数，不修改正式默认值、L4、资金、凭据或任何交易账户。

当前 L4 和既有公开数据研究均不支持把正式策略视为已证明收益能力。问题因此不是“怎样调到历史最优”，而是：保持当前退出与风险逻辑不变时，更严格的入场选择性是否能在预先固定的时间阶段、成本和方向上稳定优于当前默认与历史激进入场。

本研究只处理三个入场参数：15m 通道回看、1m 确认根数、最大追价 ATR。初始止损固定 1.5 ATR，TP1 固定 50% @ 1.5R，TP2 固定 @ 3R，最长持仓固定 4×15m。若这一较小搜索族都不能在样本外维持正的费用后结果，不继续优化止损、止盈和持仓期。

## 固定问题

在 Binance USDⓈ-M BTCUSDT 1m 公开 K 线上，按当前 15m Donchian/ATR 和单次部分止盈语义，以信号后下一根 1m 开盘成交、保守同 bar 顺序、实际 funding 和市场单成本建模，`4/12/20/32/48/64 × 1/2/3 × 0.25/0.5 ATR × LONG/SHORT` 中是否存在一个入场选择性配置，能够：

1. 在 2021–2023 开发期通过预设的样本量、费用、年度一致性和压力成本门；
2. 固定后在 2024–2025 评价期整体及逐年为正，并至少比正式默认 `20/2/0.5` 高 5 bps/笔；
3. 在 2026H1 确认期仍为正，且评价与确认合并后保持正值。

## 语义与成本

- 15m 通道包括最近闭合的 15m bar；该 bar 闭合后清空确认窗口。
- 连续 1/2/3 根后续闭合 1m 收盘必须都越过同一通道边界；最后一根还不得超过边界 ± 最大追价 ATR。
- 信号不能在同一根 bar 成交；入场使用下一根 1m 开盘并施加方向不利成本。
- ATR(14) 使用 Wilder 平滑及 previous close；所有 R 值在入场时冻结。
- 止损与两档止盈按 1m OHLC 模拟；同一分钟同时触发相反边界时按止损先发生。TP1 后仅剩余仓位继续受同一止损和 TP2 约束。
- 最长持仓 60 分钟；时间退出使用到期后首个可行动 1m 开盘。
- market entry、stop/time exit 使用 taker；TP 使用保守 taker。base 为每边 4 bps fee + 10 bps 不利滑点/点差，stress 为每边 4 bps + 15 bps，favorable 为每边 4 bps + 2 bps。
- 持仓跨越 Binance funding 事件时按实际 funding rate、方向和剩余名义计入。
- 交易在每个配置/方向内不可重叠；退出后的下一根 1m 才可重新武装。它是持续重武装代理，不是产品真实的人工激活日程。

## 数据阶段与门

| 阶段 | 时间（UTC，end exclusive） | 作用 |
|---|---|---|
| development | 2021-01-01 至 2024-01-01 | 一次计算完整 72 配置分布并按固定门选最多 3 个 |
| evaluation | 2024-01-01 至 2026-01-01 | 只评价已固定候选；不重排、不换门 |
| confirmation | 2026-01-01 至 2026-07-01 | 只评价通过 evaluation 的单一优先候选 |

开发门要求每个方向配置至少 100 笔，base 与 stress 平均净收益/笔均大于 0，三个开发日历年至少两个为正，且最差日历年不低于 -10 bps/笔。通过者按“最差年度 base 均值、stress 均值、base 均值”依次降序，固定最多三个；完全同分时依次偏好更长回看、更多确认、更小追价边界。

评价门要求至少 60 笔、base 与 stress 整体均为正、2024 与 2025 各自 base 为正，并在相同方向上比正式默认 `20/2/0.5` 的 base 均值至少高 5 bps/笔。只按开发排序取首个通过者进入确认，不因评价结果重排。

确认门要求至少 15 笔、base 与 stress 均为正，且评价+确认的 base 合并均值为正。

- `SUPPORTS_WITHIN_SCOPE`：三个顺序门全部通过。
- `DOES_NOT_SUPPORT`：数据和样本量充分，但没有开发候选，或固定候选在评价/确认的 base 或 stress 均值不正。
- `INSUFFICIENT_EVIDENCE`：主要因为样本量门、历史数据暴露或代理语义限制而不能完成判定。
- `CANNOT_DETERMINE`：数据身份、实现或关键执行语义无法核对。

## 已知暴露与限制

2021–2026H1 的 BTCUSDT 公共数据已被仓库内其他问题查看，且 20/2/0.5、最长 96×15m 的旧策略代理曾运行。本问题锁定的是此前未执行的当前 4×15m 退出与完整入场选择性矩阵，属于规则级时间评价，不是全新市场数据留出；即使通过，也只能形成待继续纸面/小额 Demo 验证的候选，不能证明 Alpha 或授权产品改动。

1m OHLC 无法重建 bar 内路径、mark price、订单簿、排队、部分成交、延迟、强平和真实个人费率；保守顺序和成本只是代理。产品是人工选择时点的一次激活，而研究代理持续重武装，两者不可冒充同一绩效。只有候选通过全部门后，才值得另行用 NautilusTrader 核对事件顺序和在线/离线一致性。

## 结论

`DOES_NOT_SUPPORT`。完整结论、反证与产物身份见 `result.md` 和 `results.json`。72 个配置没有任何一个在开发期取得正的 base 或 stress 平均净收益，因此没有固定候选，评价期和确认期均未解封。
