# TRX/PAXG 候选的选择偏差与分散逻辑审计

## 固定问题

父候选 `TRX_PAXG_USDM_MONTHLY_25PCT_EACH` 已在现货顺序阶段和 456 日 USD-M 场所迁移中通过原有门。本题不寻找新候选，而是审查：在如实计入它形成前已经查看的相关研究、资产选择、黄金强势背景和已暴露时间后，现有证据是否仍足以支持“具有稳定盈利可能、值得进入 Demo”的结论；固定 25% TRX + 25% PAXG 的月度配置逻辑究竟来自可推广的分散/再平衡机制，还是来自碰巧选中了两个上涨资产。

- 主要类型：`STRATEGY_CANDIDATE` 的证伪型复核。
- 决策用途：决定父候选维持、降级或撤回 Demo 候选资格；用户已经明确暂不进入 Demo。
- 最强允许主张：已暴露历史与外部先验是否足以继续支持产品考虑；不证明因果、未来收益或 Alpha。
- 数据边界：Binance Spot 2021-01-01～2026-06-30；其中父候选 TRX/PAXG 全部结果已暴露。BTC/ETH 同期价格也在其他研究中出现过，所有新增比较均视为已暴露证据上的审计，不称为 holdout。USD-M 2025-04-01～2026-06-30 只引用父研究，不新增选择。

## 预先固定的反解释与比较

1. **只是持有两个上涨资产**：比较相同初始 25%/25% 但不再平衡的 buy-and-hold。
2. **只是黄金牛市**：报告每条腿的收益贡献、候选去掉最佳年/月后的结果，以及 PAXG 单腿贡献。
3. **TRX 是事后挑中的加密资产**：在完全相同 25% 加密 + 25% PAXG、月度、成本模型下比较 BTC/PAXG 与 ETH/PAXG；它们是资产选择审计基准，不可事后替换为新候选。
4. **固定名义等权并不等于风险平衡**：报告逐段风险贡献，并比较外部实务中常见的 360 日逆波动、月度、总 gross 0.5 诊断。
5. **顺序分段掩盖了跨研究搜索**：在共同 2022～2026H1 窗口上，以六个固定组合返回列的 Sharpe 分布计算 3、12、33 次试验的 DSR 敏感性；12 次对应 TRX/PAXG 形成链条中三组各三个诊断/候选再加本次固定比较的保守相关试验量级，33 次是父候选形成前已完成研究问题数的上界，不把它们冒充精确独立试验。
6. **短样本与市场状态偶然**：报告年度、滚动 365 日、TRX 尾部日、90 日成块自助和六配置 CSCV/PBO 诊断。

## 审计判定

只有以下证据同时成立，父结论才可维持 `SUPPORTS_DEMO_CANDIDATE_WITH_LIMITS`：

- 共同窗口 12-trial DSR 至少 0.80；33-trial 作为上界敏感性必须披露，但不单独作为硬门；
- 固定候选相对同资产 buy-and-hold 的 Base 总收益增量为正，且成块自助中胜出概率至少 60%；
- 固定候选不被 BTC/PAXG、ETH/PAXG 或 TRX/PAXG 360 日逆波动方案严格支配（对方总收益不低、Sharpe 不低、最大回撤不深，且至少一项严格更好）；
- TRX 的 ex-post 方差贡献不超过 80%，否则“balanced”只能解释为名义配置而非风险平衡；
- 90 日成块自助下，超过 4% 年化全资本门的概率至少 80%；
- 外部资料至少支持“黄金与加密风险资产可形成时变分散”这一宽机制，并明确不存在直接支持 TRX 固定名义等权的证据。

任一硬门失败，父候选降为 `INSUFFICIENT_EVIDENCE`，暂停 Demo 考虑；不会从审计基准中挑选表现更好的替代品。
