# 15 分钟边界首分钟订单失衡的中期预测性

## 状态与用途

- 主要研究类型：`PREDICTIVE`。
- 稳定产品基准提交：`0bdfeffa616260cebd2d2188ddc8deb9e85c77f4`。
- 固定正式策略背景：`ONE_SHOT_DONCHIAN_ATR_BREAKOUT` `1.0.1`，`BTCUSDT-PERP`。本题不运行、比较或修改正式策略。
- 开题日期：2026-07-22。任何本题 Binance 1m 结果下载前固定问题、方法、门槛和代码身份。
- 用途：判断一种低延迟、公开基础数据可持续计算的订单流信号，是否值得进入独立 `STRATEGY_CANDIDATE` 研究。预测性通过不等于策略净盈利、产品可用或真实交易许可。

本题只读 Binance 官方公开历史文件和当前 Git 内研究资产；不读取产品数据库、业务数据、凭据或运行配置，不启动产品运行时，不调用交易所变更端点。新增材料只写本目录；大型原始数据与可重算中间产物放在 Git 外。

## 问题来源与候选选择

Kim 与 Hansen（2026）的最新原始研究在 Binance 六个主要 USDT 永续的逐笔成交中发现：15 分钟边界前 10 秒订单失衡与未来 4–12 小时收益存在预测关联，8 小时和 12 小时的公开状态成分四分位效应约为 9.8 bp 和 16.9 bp。但论文明确没有给出扣除执行成本的独立策略，10 秒开盘收益的可预测毛幅度又明显小于交易成本。本题因此不复述“边界有效”，而检验一个对个人项目真正关键、论文未回答的适配差异：**无需保存逐笔数据，只使用官方 1m Kline 的 taker-buy volume，延迟到下一分钟执行时，信号是否仍有足够强的增量预测性。**

先行调研比较了以下少量方向：

| 候选 | 决策价值与未解决差异 | 数据/现实成本 | 本轮决定 |
|---|---|---|---|
| 15 分钟边界首分钟订单失衡 → 8–12h 收益 | 新的周期性订单流机制；论文未检验 1m Kline 代理、下一分钟可行动性、论文外资产与净成本策略 | 单场所、单腿、公开基础数据、短反馈 | **选中** |
| 低成交活跃度条件下的短期反转 | 可能是流动性补偿，但文献效果偏向小而不活跃资产 | spread/冲击与尾部风险不利个人 | 暂缓 |
| 60 分钟配对/协整交易 | 与既有 BTC 残差反转不同 | 双腿同步成交、两侧 funding、关系再估计 | 暂缓 |
| 跨币负向 lead-lag | 有文献证据 | 净优势极薄且依赖高速自动化 | 淘汰 |
| 一般日历/时段收益 | 易取数据 | 近年大样本文献已有强反证 | 低优先级 |

Halpha 已完成的 BTC 冲击 beta-gap、BTC 中性残差反转及活动度条件化 beta-gap 均为 `DOES_NOT_SUPPORT`；本题不再改它们的阈值或币种，而转向不同的订单流机制。

## 固定问题与最强允许主张

问题：在 Binance USD-M 的 `BNBUSDT`、`LINKUSDT`、`UNIUSDT`、`FILUSDT` 永续中，UTC 15 分钟边界开始后的**完整首分钟**主动买卖量失衡，是否能在该分钟收盘后、从下一分钟 open 开始，对未来 12 小时收益提供相对普通伪边界及简单价格/成交量解释的增量预测信息？

若逐级通过，本题最多主张：在固定场所、合约、时期、1m 代理、下一分钟动作语义和 12 小时目标下，存在值得另开策略候选研究的预测信息。不得主张因果、已证明 Alpha、成本后盈利、未来长期稳定、逐笔论文已被精确复现或产品可以启用。

## 对象、数据和时间边界

- 场所：Binance USD-M USDT 线性永续。
- 资产：`BNBUSDT`、`LINKUSDT`、`UNIUSDT`、`FILUSDT`。均在 2021 年前上线；2026-07-21 市场名单中持续交易，当前属于成熟或较高活动度候选。它们不在来源论文的 BTC/ETH/XRP/SOL/DOGE/ADA 六币样本中。
- 原始数据：Binance 官方月度 1m Kline ZIP 与同目录 `.CHECKSUM`。字段仅使用 open、close、base volume、quote volume、taker-buy base volume、trade count。时间戳完整性扫描发现官方 checksum 有效的 `FILUSDT` 2022-02 月文件缺最后 3 日、2022-04 月文件缺最前 2 日；在任何结果计算前固定以对应 5 份官方日文件及 checksum 补齐，并要求合并后严格等于完整分钟网格，不做价格填充。
- 订单失衡代理：`oi = 2 * taker_buy_base_volume / base_volume - 1`，无成交量时未知。它是分钟聚合代理，不是论文的前 10 秒逐笔订单失衡。
- 时区：UTC。只有边界首分钟已经收盘后才允许形成信号；目标从下一根 1m bar 的 open 到其后 12 小时窗口末尾 close。

## 事前固定方法

每个资产和事件分钟 `t`：

1. `oi_t` 使用已收盘边界首分钟 Kline 的 base volume 与 taker-buy base volume。
2. `y_t = log(close[t+720] / open[t+1])`；最后没有完整 12 小时目标的事件丢弃。
3. 主回归只取真边界 `minute ∈ {0,15,30,45}`：`y ~ const + oi + current_1m_return + prior_15m_return + log1p(quote_volume)`。
4. 每个资产报告 `oi` 从样本 25% 分位到 75% 分位对应的预测收益变化（IQR effect，bp）；四资产等权平均为主效应。
5. 推断使用同一组 UTC 日历块跨四资产联合抽样、长度 4 日的 moving-block bootstrap，固定 999 次、种子 `20260722`，每次重新估计回归，保留重叠目标与跨资产依赖。
6. 固定伪边界为相对真边界偏移 `+2`、`+4`、`+7`、`+11` 分钟；主反证为真边界四资产平均 IQR effect 减去四个伪边界均值。
7. 固定稳健性：8h 目标、额外延迟至第 5 分钟 open、只保留 15/30/45 分钟（排除 top-of-hour）、排除 00:00/08:00/16:00 funding 开口、订单失衡只用符号、按资产和自然年分解。它们只用于反证，不用于改选主配置。

本题没有策略动作，因此不创建 VectorBT Portfolio。pandas/numpy 用于时间对齐和批量 bar 特征，statsmodels 用于 OLS，VectorBT `1.1.0` 固定研究环境身份。若预测门通过，后续策略题才用 VectorBT 在唯一时间轴上处理重叠信号、仓位、费用、funding 和成交代理。

搜索披露：4 个固定资产、1 个主配置、4 个固定伪边界、5 类固定稳健性、2 个辅助目标/延迟、资产与年份分解。不得查看结果后改资产、边界相位、目标、控制项、成本线或门槛。

## 事前门槛与否定条件

开发门全部满足才启封 evaluation：

1. 数据质量 `PASS`；四资产各阶段完整分钟网格，主事件合计至少 100,000 个；
2. 四资产平均 12h IQR effect `> 0`，联合 4 日 block-bootstrap 95% CI 下界 `> 0`；
3. 真边界减四个伪边界平均的差异 `> 0`，95% bootstrap CI 下界 `> 0`；
4. 四个资产的主效应均 `> 0`，development 的 2021、2022 年均 `> 0`；
5. 延迟 5 分钟、排除 top-of-hour、排除 funding 开口后三个效应均 `> 0`；
6. 主平均 IQR effect 至少 `12 bp`，只作为值得消耗留出数据的有利 round-trip 成本相关线，不代表已经模拟费用或可成交净收益。

任一情况构成强反证或停止理由：主效应非正或 CI 跨零；伪边界同样强；仅单一资产/年份成立；延迟后消失；top-of-hour 或 funding 时点完全驱动；控制简单动量和成交量后消失；幅度低于 12 bp。开发失败时不下载 2023/2024；不得以临近阈值为由换币、改相位或改持有期。

本题结论只能为 `SUPPORTS_WITHIN_SCOPE`、`DOES_NOT_SUPPORT`、`INSUFFICIENT_EVIDENCE` 或 `CANNOT_DETERMINE`。预测性通过只释放一个策略候选研究问题，不产生产品交接包，也不改变正式策略、L4、资金或真实账户状态。

## 实际结果

development 数据质量通过，共 280,091 个主观察，但主 12h IQR effect 为 **−3.108 bp**，4 日联合 block-bootstrap 95% CI 为 **[−6.631, +0.413] bp**；真边界还比四个伪边界平均低 2.160 bp。2021 为负、四资产中三项为负，8h、延迟 5 分钟、排除整点、排除 funding 和 sign-only 均为负。除数据质量和样本数外的全部事前门槛失败。

结论：`DOES_NOT_SUPPORT`。2023 evaluation 与 2024 confirmation 没有下载或查看；本题不释放策略候选。完整机器结果见 `development.json`，解释与复现见 `result.md`，数据身份见 `source_manifest_development.json`，实际失败与修复见 `attempts.md`。可重演入口为 `study.py`。
