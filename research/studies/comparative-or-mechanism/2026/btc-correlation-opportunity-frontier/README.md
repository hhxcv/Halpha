# BTC 相关性可盈利机会前沿审计

## 问题与结论口径

问题：在 Halpha 当前“个人维护、小资金、基础公开价格/成交量数据、下一可行动时间、保守成本、不碰最低活动高风险币”的边界内，现有证据是否支持新增一个由 BTC 相关性直接导出的策略候选？哪些方向已经关闭、只是风险用途，或必须等新数据/新条件？

最终结论：`DOES_NOT_SUPPORT`。

这不是“市场不存在任何 BTC 相关 Alpha”的全称断言。它只表示当前直接可研究、与项目适配的价格型机会没有达到候选门槛；不支持继续用近邻阈值、单币、方向或更差活动层搜索。结论不影响既有正式策略，也不否定 funding carry 等不同经济机制。

## 为什么可称“当前空间基本完成”

本轮不是枚举所有数学模型，而是覆盖了 BTC 相关性最直接、最能由基础数据证伪的三条收益路径：

1. BTC 先动、山寨币稍后跟随——在成熟和中等活动两层、下一 open、多个窗口/阈值下均只有约 2 bp，额外延迟消失。
2. 山寨币相对 BTC 走过头、随后回归——小时级主均值为负，并暴露单边行情左尾。
3. 共同趋势/相对强弱——历史 momentum 研究已有失败/不足，且正式策略已提供固定 trend 比较基准。

再继续最容易产生的不是独立信息，而是：从已看结果选择正冲击、负 residual、DOGE/NEAR/ANKR、止损或某个 5m/4h 窗口；这属于选择偏差。更复杂 PCA/copula/ML 也不会自动创造毛幅度，近期 crypto pairs 证据反而显示成本后为负。

## 当前最合理的项目行动

- 不新增第二个产品策略，不改正式策略/L4/资金/真实账户。
- 保留三个问题的代码、全部配置、manifest、失败和封存状态，后续查重先引用。
- `btc-market-relationship-monitor` 继续作为描述/风险页面，不展示“机会分数”或盈利暗示。
- 研究资源转回不同经济来源，而不是 BTC 相关性近邻：已有 funding carry 候选按其自身限制维护；新的策略题应先证明独立机制与净成本余量。
- 若未来产品明确选择多资产策略，再开“BTC beta/correlation 是否改善 sizing/风险限制”的 risk-model 问题；它不应预先包装为 Alpha。

## 重新打开的必要条件

只有至少一个真正新条件出现才重开相关方向：

1. 可合法持续获取并留存 point-in-time 历史 L1/L2、真实 spread/price-impact、跨所同步时钟或订单流，足以建模低活动/1m 执行；
2. 新增独立市场时期或新场所结构，并在开题前固定规则，而不是复用 2024 子组；
3. 出现可解释、可对冲且长期稳定的结构关系（不是纯相关）或产品已选中的多资产风险需求；
4. 实际成本/延迟能力发生可验证的数量级改善，使 1–3 bp 毛效应不再先验无意义。

否则，本轮结论用于阻止重复研究，不是等待换参数的临时暂停。

## 数据与复现

三项新研究共同保留 420 个唯一官方月 ZIP、144,160,250 bytes；所有文件在 Git 外缓存，Git 内有官方 URL/checksum/实得 hash。两个 lead-lag 研究各自独立复算，residual 研究复用并重新验证同一缓存。`audit.py` 从 checkpoint 记录的基准提交读取并校验当时的 L4，同时校验 9 个关键研究证据文件和两个源 manifest 后生成 `audit.json`；当前产品 L4 与研究基线的差异仅记录为漂移，不会改变历史研究身份或造成误失败。
