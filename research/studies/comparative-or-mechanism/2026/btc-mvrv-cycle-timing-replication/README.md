# BTC MVRV Z-score 周期择时独立复核

## 问题与用途

Grobys、Näsman、Sandretto（2026）报告 MVRV Z-score、NUPL 和 CVDD 在 2013～2025 三个 BTC 周期中优于 buy-and-hold。本题不搜索链上指标，而复核其最强、最简单的 MVRV Z-score 结论：在独立的 Coin Metrics 数据、无未来信息的扩展标准差、下一日行动、现实成本和公平同起点基准下，固定 `Z < -0.2` 入场、`Z >= 5/6/7` 退出是否仍能识别 2015/2018/2022 低点与 2017/2021/2025 高点；其优势究竟来自周期信息，还是基准起点和事后阈值。

- 类型：`COMPARATIVE_OR_MECHANISM`；不作为产品策略候选。
- 决策价值：判断链上持仓成本能否为下一项 BTC 策略提供独立的大周期状态，而不是继续调整价格趋势参数。
- 最强主张：链上 MVRV 与大周期转折存在可重复的时序一致性；没有识别设计，不声称因果或未来盈利。
- 产品作用：`NONE`；不进入 Demo。

## 固定定义

Coin Metrics 提供 `CapMVRVCur = market cap / realized cap` 与 `CapMrktCurUSD`。因此：

`realized_cap_t = market_cap_t / MVRV_t`

`MVRV_Z_t = (market_cap_t - realized_cap_t) / expanding_std(market_cap_0..t)`

扩展标准差至少需要 365 日，禁止用完整样本标准差形成可行动信号。完整样本标准差只作“常见图表是否隐含未来信息”的诊断。

论文三个固定变体全部保留：入场均为 Z 从上向下穿过 -0.2；退出分别为从下向上穿过 5、6、7。信号在日数据完成后形成，下一日价格行动；每单位换手 Base/Stress 成本 25/50 bp，现金收益为零，不做空、不加杠杆。三个变体不排序、不选冠军，也不能从复核中直接晋升为候选。

## 顺序与基准

- warmup：2010-07-18～2013-12-07。
- development：2013-12-07～2022-01-01，覆盖 2015/2018 低点和 2017/2021 高点。
- confirmation：2022-01-01～2026-07-01，仅在 development 保留跨周期机制时启封，覆盖 2022 低点、2025 高点和之后熊段。
- 主基准：与链上规则完全相同的起止日、成本和初始资本下持续 1x buy-and-hold。
- 严格牛段诊断：在预先固定窗口内取事后低点和高点；候选使用当时真实持仓状态，buy-and-hold 被假定可在低点后第一日完美买入。
- 论文口径诊断：2013-12-07 开始持有；只用于解释原论文结论，不替代主基准。

周期搜索窗与上一 BTC 宏观研究一致：2014-12～2015-03 / 2017-11～2018-02，2018-11～2019-03 / 2021-10～2022-01，2022-10～2023-02 / 2025-09～2025-12。端点只评分，不进入信号。

## 反证门

Development 只要满足以下条件就允许查看 confirmation：

- 至少两个退出阈值的同起点 Base 总收益高于 buy-and-hold，且最大回撤更浅；
- 三个变体对 2015/2018 两个低点的最近执行入场均在 ±180 日内；
- 至少两个变体对 2017/2021 两个高点的最近执行退出均在 ±180 日内；
- 5/6/7 三个阈值结论方向一致，不依赖唯一阈值。

Confirmation 要求：2022 低点入场和 2025 高点退出分别在 ±180 日内；至少两个阈值在同起点 Base 下继续优于 buy-and-hold；严格 2022 低点到 2025 高点诊断不得把“风险调整更好”误写成原始收益超越。若 2025 顶部没有触发退出，必须明确判定固定历史阈值发生衰减，不能用较低新阈值补救。

无论结果如何，本题不证明当前已经见底，也不把链上数据运行依赖引入产品。
