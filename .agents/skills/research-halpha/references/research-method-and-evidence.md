# Halpha 研究方法与证据

## 结论强度

研究投入随结论用途增加：

| 用途 | 最小证据 |
|---|---|
| 探索 | 问题、数据范围、最小实验、明显限制和不能推出的结论 |
| 比较证据 | 固定比较对象、现实成本、时间顺序、主要反例、参数或样本敏感性和可重演命令 |
| 产品考虑 | 前述全部，加上独立时间评价、搜索范围披露、市场状态变化、执行可行性和明确失效条件 |

不要为了形式把探索包装成确认研究。较弱但诚实的结论优于复杂而失真的正向结论。

## 现有研究与外部资料

选题前先读取项目当前事实、相关设计和已有研究，再调研与候选问题直接相关的外部工作。扫描 `research/**` 时至少按机制、instrument 或 venue、问题、数据期间和结论判断重叠；已有研究已经回答或覆盖主要差异时，优先引用、重演或扩展，不另做同义问题，也不为去重建立全局索引或登记表。

问题必须能说明当前决策或信息价值，以及相对现有工作仍未回答、需要复现或需要适配的差异；一般新颖、容易实现或可能产生漂亮指标不构成价值。候选还要适合一人维护，符合用户已明确的资金与验证周期限制，并把未来运行复杂度和研究成本一起计入。依赖规模资本、跨场所库存或长周期验证的方向，只有直接影响当前决定时才值得优先研究；未明确的个人资金规模保持未知。

资料优先级为：原始论文；交易场所、数据源和标准的官方规则或接口文档；研究库的官方文档与源码；可核查的独立复现。二手文章、论坛和聚合结果可以提供线索，不能单独支撑关键机制、接口或经济结论。

对实际采用的资料记录链接、发表或访问时间、所解决问题、关键假设、版本或数据范围、适用于当前问题的理由和未覆盖差异。市场、费用、接口和库行为可能变化时，以当前官方来源核对。外部结论不能直接移植；先验证定义、时间、市场、数据、成本和执行条件。

遇到影响实现或判断的困难时，先定向查找原始研究、官方文档、源码和已记录问题，再决定是否自制最小补充。记录采用或放弃某个答案的理由；没有可靠答案时保留未知或降低结论，不用未经验证的变通方案掩盖问题。

## 数据与时间

- 固定 instrument、venue、contract、bar/tick 含义、timezone、起止边界和 cutoff。
- 检查缺口、重复、乱序、修订、幸存者偏差和不可获得的未来字段。
- 训练、选择、调参和评价按事件时间分开；重叠区间不能假装独立。
- 记录所有已经查看的评价区间。查看后只能继续作为已暴露证据使用。
- 数据源变化或历史修订时，保存内容摘要、版本或足以重新取得同一输入的规则。

### 留出启封

在首次查看拟作为未暴露评价区间的结果前，用现有研究说明、代码快照或摘要保存一个轻量检查点，至少固定问题、已搜索范围、开发或选择门槛、输入身份、方法或代码身份、允许修复以及启封规则。它用于解释后来哪些变化仍属于原问题，不要求固定 manifest、schema 或单独跟踪系统。

任何可能显示方向、收益或相对表现的图表、摘要和人工浏览都算暴露。只检查文件可读性、字段完整性或哈希且不能看到结果的访问可以标为完整性检查，并说明为何没有泄露结果。

开发门槛失败时默认停止，让留出区间保持未暴露。只有“确认拒绝”本身会改变当前决定时才继续启封；一旦使用，就按已暴露证据记录，不能在后续问题中重新称为独立确认。

## 经济现实

按研究对象纳入：

- maker/taker 手续费和费率变化；
- 永续合约资金费；
- spread、slippage、延迟和成交时点；
- 最小数量、精度、杠杆、保证金和清算约束；
- 流动性、容量、部分成交和无法按收盘价成交；
- 无交易、简单持有、简单规则或当前策略等合理基准。

缺少关键成本时可以研究机制，但必须禁止成本后盈利或可交易结论。

当前正式策略不是每项研究的强制基准。只有能在独立研究边界内公平重演、或存在决策相关且明确标注差异的有界代理时才比较；不得为了表面可比性在研究侧维护第二份产品策略实现。无法得到相称比较时，保留“未知”或“未运行”，并用更简单、可解释的基准回答能够回答的部分。

## 搜索、稳健性与反证

披露尝试过的策略族、特征、参数、市场和指标。搜索越广，最佳结果越可能来自偶然；不得只保存胜者或把人工浏览排除在搜索次数之外。

根据用途选择最少充分检查：

- 相邻参数和合理替代定义；
- 多个连续时间段或 walk-forward；
- 趋势、震荡、高低波动和流动性变化；
- 单一时期、单一交易或单一方向支配；
- 成本压力和延迟压力；
- 更简单机制或数据泄漏解释。

样本不足、候选过多或独立区间已经暴露时，结论应降为 `INSUFFICIENT_EVIDENCE`，而不是放宽门槛或寻找新的有利指标。

## 最小研究记录

研究说明至少能回答：

1. 问题来自哪个当前项目缺口，有什么用途，现有工作留下了什么差异？
2. 输入来自哪里，时间范围和 cutoff 是什么？
3. 哪些外部资料实际影响了选题或方法，为什么适用？
4. 运行命令、环境和输出文件是什么？
5. 实际尝试过什么，失败和条件变化是什么？
6. 成本与成交假设是什么，哪些未建模？
7. 结果对时间、参数、成本和市场状态是否敏感？
8. 最强支持、最强反证和当前结论是什么？
9. 哪些证据已暴露，什么新证据才可能改变判断？

涉及留出评价时，研究记录还应能指出启封前固定了什么、何时启封以及启封理由。完成前核对小型输入、命令、结果与采用的摘要仍可取得；外部大型数据记录来源身份、重取规则、预期位置和本地缓存是否可保留或重新取得。交付时说明研究文件是否已被 Git 跟踪；未跟踪材料只能称为留存在当前工作树，不能称为已有持久 Git 历史。

结果文件保留原始指标和样本计数；说明文件解释含义。图表用于揭示分布、时间变化和异常，不代替数值与可重演输出。
