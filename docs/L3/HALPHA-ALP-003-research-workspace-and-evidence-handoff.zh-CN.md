# 0. 设计结论【ALP-RSCH-SUM-001】

Halpha 把候选研究放在交易产品之外的独立研究工作区。研究工作区可以独立安装研究依赖、执行探索、保存失败结果并按需启动本地研究界面；它不属于 App、Executor、产品数据库、场所写链或产品发布组，也不取得真实交易凭据。

在适用 L4 一次性固定的研究政策、评价口径、数据边界、搜索与资源最大预算和停止条件内，外部 AI 是主要研究工作者：它自行形成并预登记具体研究任务和可证伪用例、调用确定性 CLI、筛除不合格候选、组织反证、复现和最终候选材料。项目所有者不必逐项批准任务、中间运行或淘汰，只对达到固定门槛的最终候选作必要结果审查。AI 的生成、评价、复核和排序始终是派生材料；硬门槛、统计结果、数据事实、经济证据判断、正式策略晋升和任何产品或真实资金决定不由 AI 拥有。

研究工作区只交付可重复检查的研究证据包。研究证据包是经济证据判断的材料，不是经济证据判断、正式策略、交易计划、资格证明、资金权限或动作命令。研究结果为正、程序运行成功或研究界面显示“通过”，都不能自动晋升策略或改变产品行为。

候选研究实现与正式策略实现分离。候选研究可以使用只在研究环境中存在的统计、数据处理和可视化方法或成熟组件，但不得用第二量化框架承担 HALPHA-ALP-002 与 HALPHA-ENG-002 已交给 NautilusTrader 的指标、历史回放、撮合、绩效或策略生命周期。研究实现和输出不能成为第二套 Halpha 正式回测、资格验证或实盘运行栈。只有显式晋升后进入交易产品唯一正式源码、静态策略清单和 HALPHA-ALP-002 同源验证路径的实现，才可以成为正式策略依据。

长期组件选择固定为：NautilusTrader 承担唯一策略指标、历史回放、撮合、费用、绩效统计和报告数据来源；NumPy、pandas、PyArrow 与 Pandera 承担表格、Parquet 和输入质量；SciPy 承担预声明的重采样、假设检验和多重检验调整；Plotly 只把已有结果生成可离线打开的 HTML；Pydantic 与 Python 标准库 `json`/`hashlib` 形成规范清单、请求、回执和摘要；JupyterLab 与 Jupytext 只提供可删除的交互创作界面。精确版本、锁摘要、许可证和目标平台资格属于适用 L4，不能由本 L3 推断当前可安装或可用。

可引用运行只从一个无 Notebook 隐藏状态、由人和 AI 共用的命令行入口在新进程中执行；每个回测运行使用独立进程和独立输出目录。AI 编写的 study 是未受信输入，不能与受信 runner/NautilusTrader 引擎处于同一解释器或能力边界：受信父进程唯一持有数据文件、时钟、NautilusTrader 引擎、指标/状态、订单与成本模型、确认性计数和 canonical artifact writer；模型外沙箱中的 study worker 只按事件取得规范 point-in-time observation，并返回 schema 受限的 research intent。Pydantic、路径校验、venv 和普通新进程本身都不是安全隔离；HALPHA-ENG-003 v0.3.0 与适用 L4 还必须资格化文件、网络、秘密、进程树、输出和资源强制边界。JupyterLab 可以单独按需启动；不启动、进程失败或从未来研究环境移除时都不得影响同一命令行运行重演。Halpha 不调用模型供应商 API，不把 AI 客户端或 agent framework 放入产品或研究依赖，不保存供应商会话状态；外部 AI 只通过受限文件和 CLI 回执工作。本契约不建立研究 API、MCP 服务、常驻研究服务器、向量库、实验数据库、任务调度器或产品内研究页面；交易产品与研究之间唯一必要对接是项目所有者最终选择一个冻结文件证据包后进入正式策略实现与产品内重验。

# 1. 研究工作区责任与隔离【ALP-RSCH-BND-001】

## 1.1 允许的研究责任【ALP-RSCH-BND-001-REQ】

研究工作区只承担以下责任：

- 把用户直接明确或经 CTX 路由、并满足 ALP 进入要求的问题转成可重复的研究运行；
- 在已固定研究任务允许的方向、搜索预算和停止条件内，让外部 AI 生成并预登记研究用例、调用同一确定性 CLI、解释机器结果、筛除候选并组织反证、复现和最终候选材料；
- 引用带来源、时间范围和摘要的数据输入，记录数据缺失、修订、代理和许可限制；
- 执行探索、比较、反例、敏感性和可证伪检查，并保留失败、证据不足和无法判断结果；
- 生成供项目所有者最终审查和正式策略晋升使用的研究证据包；
- 删除可重建的中间缓存、图表和临时输出，不把工具内部状态提升为 Halpha 产品事实。

研究工作区不得承担以下责任：

- 写入产品数据库或读取产品数据库私有表；
- 导入 App、Executor、产品 composition root、数据库 repository、秘密加载器或场所写端口；
- 取得 DEMO/LIVE 交易写凭据、形成 ExecutionAction 或调用交易场所写方法；
- 作为交易产品启动、停止、核对、恢复、保护或发布的同步依赖；
- 让 AI 在看到验证或最终留出结果后改变原研究问题、评价口径、搜索预算、硬门槛或候选规则；
- 把 AI 自评、同源 AI 交叉赞同、聊天记录、提示词或模型置信度当作独立证据、事实或正式经济证据判断；
- 让 AI/study worker import、monkeypatch 或取得 runner、NautilusTrader 引擎/报告器、canonical artifact writer、数据路径、未来事件或确认性计数；
- 自动修改正式策略源码、策略清单、交易计划或 BuildManifest；
- 以“研究兼容层”长期维护与正式策略等价的第二实现。

## 1.2 业务隔离后果【ALP-RSCH-BND-002-REQ】

Notebook、本地命令行、本地研究界面或外部 AI 客户端只创作和消费同一文件型研究材料，不形成受支持的交易产品交互入口、交易责任、授权结果或事实权威。AI 客户端、模型、供应商会话或网络不可用时，项目所有者可以恢复性重演已经固定的请求，但不承担常规中间研究；无法重演只使本次研究保持失败、证据不足或无法判断，不能改变交易产品的启动、保护、核对、恢复、发布或既有开放责任。

本文件唯一拥有未受信研究对证据资格的业务后果，以及 observation/intent、确认性暴露与最终候选的研究语义；HALPHA-ENG-003 v0.3.0 唯一拥有工具侧父进程/worker、执行沙箱、路径、依赖、资源和凭据隔离的工程方法，适用 SYS 规范继续提供通用进程、资源与启动停止约束，适用 L4 只记录精确选择和资格证据。若这些边界不能用恶意源码反例证明 worker 无法读取未来/数据文件、污染父进程/引擎/报告器、越权或外传，AI 只可使用公开且允许模型处理、合成或脱敏材料做 discovery，不得取得确认性 holdout 或最终候选资格；不得用逐次人工代跑伪装满足“AI 主导研究”。需要 Halpha 调用模型 API、长期常驻进程、产品数据库、共享真实凭据、远程入口或对外网络服务时，本契约不再适用，必须先复核 HALPHA-ARC-001；若同时形成受支持的产品交互、自动研究到计划交接或要求用户执行代码，还必须复核 HALPHA-FLOW-001 与 HALPHA-VIS-001，再由相应 ALP、TRADEPLAN、SYS、UX、ENG 和适用 L4 形成并接受对应设计。

# 2. 研究输入、运行清单与证据包【ALP-RSCH-EVD-001】

## 2.1 研究政策与 AI 研究任务清单【ALP-RSCH-CAM-001】

### 2.1.1 研究政策清单【ALP-RSCH-POL-001-DEF】

研究政策清单是项目所有者在适用 L4 一次性选择并授权的规范 JSON 运行边界。它声明允许的方向族、允许消费的 DAT-qualified immutable `data_snapshot_ref`、对应 DAT qualification evidence 与 snapshot digest、已经资格化的规范 UTC 半开区间、统计方法族、不可放宽硬门、任务字段允许区间、单任务及累计资源上限、确认性决策族、各决策族可包含的经济主张族、确认性主张类型、全局错误率预算、数据暴露规则、holdout 规则和停止条件。它不包含经济证据判断、正式策略选择或资本权限；项目所有者改变允许方向、方法、数据可见性、数据快照、最大预算或权限时必须形成新的政策版本，AI 不得修改或扩大当前政策。

研究政策引用的 `data_snapshot_ref` 必须来自 DAT 资格材料，并附带由该材料负责的来源、许可、取得时间、`information_as_of`、事件时间、修订关系、摘要与规范 UTC 半开区间。ALP 只把该引用作为不透明、不可变、只读输入消费，不定义其来源身份、修订等价、时间归一化或追溯算法；AI、worker、任务和请求都不得创建、改写或把任意路径、名称、digest 解释为新引用。

适用 L4 固定精确研究政策版本、允许范围与上限、AI 客户端/角色边界、并发和用于资格化/试点的具体任务；此后各次 AI 自主研究的具体市场、数据区间、参数、阈值与预算消耗记录在任务清单和证据包，不要求把每个动态任务改写进 L4。政策更新属于建设/运行边界决定，不是日常 campaign 的人工审批。

`confirmation_decision_family` 是在读取确认性数据前固定的最终统计选择边界：所有会竞争同一次项目所有者选择、使用同一 holdout 或可能在该次选择中合并比较的候选都必须属于同一决策族，并共享全局错误率预算。经济主张族是决策族内部按机制和可证伪预测组织的子集，不能被用来分割统计预算或重置数据暴露。二者都只是研究统计边界，不是正式经济证据判断、策略 registry 或交易计划。

### 2.1.2 AI 研究任务清单【ALP-RSCH-CAM-001-DEF】

AI 研究任务清单是一次有界研究活动的规范 JSON 文件。它把已经由用户明确或经 CTX 路由进入 ALP 正式研究的问题，收缩为 AI 可以无需逐项人工批准而执行的范围；它不是新的 L2 研究问题、交易计划候选、资金决定或 AI 授权。至少记录：

- 正式研究问题引用、决策用途、允许研究方向与明确排除方向；
- 事前基准、主要经济机制要求、事实截止点、适用与不适用对象；
- discovery、selection/validation 与最终 holdout 的时间边界，以及标签重叠时的 gap/embargo；
- 每项派生输入、对象池、特征、归一化、缺失填补、成本校准和阈值的 `information_as_of`、fit 区间、最大回看、标签作用区间与逐 fold 重建规则；
- 候选数量、参数组合、适应性修订、最终 holdout 使用次数、墙钟、CPU/内存、输入输出体量和 AI 供应商成本预算；
- 主要比较指标、探索性与确认性统计的区别、`confirmation_decision_family`、纳入该最终选择的全部经济主张族、确认性主张是完整通过集合的 set-FDR 还是个体可晋升的预选单假设/strong-FWER、有效 p 值构造与全局 α/FDR/FWER 预算及方法、经济与现实硬门槛、压力情形和停止条件；
- AI 研究角色、各角色允许读取和写入的材料、机器协议版本、可信父进程 harness 版本、`PointInTimeObservation`/`ResearchIntent` schema 版本与摘要、允许 observation 字段/指标、允许 intent profile 与数值范围，以及逐消息数量、大小、响应时间和运行总预算；
- 项目所有者最终审查所需内容。

外部 AI 可以在尚未读取目标结果时，按当前政策自行选择少量方向、生成并冻结具体任务清单；`validate` 必须确定证明该任务是研究政策的子集。任务未扩大数据可见性、方法、预算或权限且校验通过时，不要求项目所有者逐任务批准；超出政策时只允许拒绝或等待新的政策/授权。任务清单只能在尚未读取相应结果时形成或修订；任何实质修订必须产生新的 `registration_revision`、明确耗用的搜索预算和新任务版本，不能覆盖旧清单或继续使用已经暴露的最终 holdout。研究政策和任务都不能保证产出候选。

## 2.2 研究用例预登记与搜索全量保留【ALP-RSCH-REG-001-REQ】

每个进入确定性运行的研究用例必须先在干净 Git commit 中固定研究用例身份、经济机制、可观察预测、最强替代解释、可证伪条件、可用输入的决定时点、基准、成本、参数及其有限范围、数据切分、主要指标、硬门槛和预期产品差距。对象池、特征选择、归一化、缺失填补、成本估计、阈值和超参数等全部拟合态都必须记录其 `information_as_of`、fit 区间、最大回看与标签作用区间，并在每个时间 fold 只用当时可得过去数据重建；gap/embargo 至少覆盖信息回看和标签跨界范围，横截面对象池必须是 point-in-time。没有可证伪机制、依赖决定后信息、无法给出成本后比较对象或需要突破当前数据/动作能力的用例，在读取结果前即被拒绝或明确等待。

同一任务内全部提案、参数尝试、修改、运行、淘汰和失败通过不可变用例文件、运行回执和冻结时生成的有序索引全量保留；不建设实验数据库或追加式业务账本。AI 不得只登记最终胜者、把参数变化重命名成新机制，或删除不利运行后重算“尝试次数”。预算外的新方向或看到验证结果后的规则修改必须进入新的任务版本；看到最终 holdout 后修改规则的候选只能结束，不能在同一 holdout 上再次取得正向资格。

discovery/selection AI 只能获得任务声明的 discovery 与 validation 输入；这些区间参与假设生成、调参或筛选后，其统计只具有探索性淘汰含义。进入确认性最终 holdout 前，必须一次性冻结一个 `confirmation_decision_family` 中所有可能竞争同一次最终人工选择、共用该 holdout 或可能在选择时合并比较的经济主张族、候选/假设、规则、参数、源码、有效 p 值构造、全局错误率预算、调整方法、门槛和选择理由；不能把相关候选拆成多个经济主张族分别取得较宽松的确认性门槛。holdout 由新的只读角色上下文和沙箱进程执行，该上下文只能取得冻结候选、允许的 holdout 执行入口和受限机器回执，不能读取行级 holdout、改写候选源码或清单。相同模型或供应商可以承担不同角色，但这种新上下文只提供程序性隔离，不构成独立经济证据。

每次确认性使用必须进入不可变确认性暴露索引，并记录 `confirmation_decision_family`、`confirmation_exposure_key`、DAT-qualified immutable `data_snapshot_ref`、DAT qualification evidence 引用、snapshot digest、规范 UTC 半开区间、实际读取摘要、修订说明以及该 holdout 上全部被检验假设和结果，而不是只记录胜者。可信父进程只能解析研究政策允许的 `data_snapshot_ref` 并核对其资格证据、摘要和区间；引用缺失、变化、修订或重导出关系不明、区间不可比较，或请求与资格材料不一致时，必须在打开 holdout 前拒绝。

### 2.2.1 确认性暴露键【ALP-RSCH-EXP-001-DEF】

`confirmation_exposure_key` 是 ALP 用于判断一段确认性数据是否已经暴露的稳定查询身份，由 DAT 资格材料提供的不透明 `data_snapshot_ref` 与其规范 UTC 半开区间组成。它只拥有“相同已资格化快照的哪些区间已被确认性读取”的研究语义，不定义数据来源身份、质量、修订等价、时间归一化或追溯关系；这些仍由 DAT 拥有。

`gate` 必须按 `confirmation_exposure_key` 跨全部决策族查询相同引用的重叠、包含或被包含区间；`confirmation_decision_family` 只确定共同错误率预算，不能成为跳过既有暴露的查询分区。同一引用的已暴露区间再次使用时自动降为 discovery。新的或变化的引用不得自动视为未暴露：必须先取得新的 DAT 资格材料并由适用 L4 与研究政策显式选择；关系不明时确认性能力 fail-closed。本契约不自动判定多个来源、修订、重导出或切片之间的等价关系。

Git worktree 内的索引副本不是确认性准入权威。适用 L4 必须在单一目标宿主、单一文件系统上固定一个位于所有 worktree 与可删除 `runs/` 之外的耐久 `confirmation_exposure_root`；它是无服务、无数据库的文件控制根，只有可信父进程可写，请求、worker、AI 角色和普通研究工作包都不能选择、覆盖或重置它。每个 holdout `run` 必须在读取任一 holdout 数据字节前完成同一串行准入：先在不挂载或打开 holdout payload 的情况下解析政策允许的 `data_snapshot_ref`，核对 DAT qualification evidence、snapshot digest 与规范 UTC 半开区间，并准备 worker；取得该根的宿主级排他锁；验证当前索引代摘要；跨全部决策族查询相同引用的重叠、包含或被包含区间；在同一锁内写入包含 claim ID、请求摘要、决策族、`confirmation_exposure_key`、`data_snapshot_ref`、资格证据引用、snapshot digest 和规范 UTC 半开区间的不可逆 `EXPOSURE_STARTED`，刷新并以同文件系统原子替换形成新的不可变索引代；只有写入成功后父进程才可打开 holdout。锁、摘要、资格核对、刷新、原子替换或索引完整性任何一项失败或结果不确定时都不得打开 holdout。

`EXPOSURE_STARTED` 一经耐久写入即永久按已暴露处理，即使 worker/父进程随后失败、超时、被终止、没有生成结果或宿主崩溃；完成结果只能在再次取得同一锁后写成引用原 claim 的新索引代，不能删除、覆盖或降级原 claim。独立重演必须引用原 claim，只复核同一次暴露且不增加证据强度。排他锁只串行化短暂的确认性准入与索引更新；互不冲突的已登记运行在资源边界允许时可以并行执行，discovery/validation 不写该根并继续跨 worktree 并行。

一个确认性决策族必须共享一个预声明的全局错误率预算和方法，覆盖该次最终选择中全部经济主张族与假设。BH/BY 的 FDR 结论只属于完整拒绝/通过集合；在看到 holdout 后从集合中再排序、抽取一个或少量候选，并不使被抽取候选自动取得同一 q 值或个体错误保证。要把某个候选标为可单独晋升的确认性最终候选，必须在任何 holdout 暴露前二选一：先按不使用该 holdout 的固定规则只预选一个候选，再对该单一假设使用全局 α；或对全部可能晋升假设使用预声明且资格化的 strong-FWER 门，最低基线为有限族 Bonferroni。只通过 BH/BY set-FDR 的集合可以作为集合级研究结果，但其中任何事后选出的成员都不能直接进入正式策略晋升；它必须在新的未暴露数据上完成上述个体确认。更复杂 selective-inference 方法不属于当前基线，需先修订本 L3 并资格化。

多个原本独立的决策族如果使用同一数据后又被合并、排序或竞争同一次最终晋升，其既有确认结果不得合并宣称；只有事前声明并资格化的分层检验/预算分配，或新的未暴露 holdout，才能形成该合并决策的确认性资格。若未来需要连续或自适应确认，必须另行修订本 L3 并资格化相应方法，不能把 BH/BY、新任务编号或新决策族名称当作通用修补。

## 2.3 研究运行清单【ALP-RSCH-EVD-001-DEF】

研究运行清单是一次可独立重演研究运行的文件型说明。它不是产品数据库记录，也不授予研究结果权威性。每次进入可引用研究证据包的运行至少记录：

- 正式研究问题、`confirmation_decision_family`、经济主张引用、预期决策用途和可证伪条件；
- 研究政策、AI 研究任务、用例、`registration_revision`、运行阶段、角色协议、`confirmation_decision_family`、经济主张族、`confirmation_claim_type`、全局 α/FDR/FWER 预算和适用搜索预算引用；
- 研究源码修订、未受信 worker 源码摘要、研究依赖锁摘要、可信父进程 harness 版本、运行入口和非秘密配置摘要；
- `PointInTimeObservation`/`ResearchIntent` schema 版本与摘要、允许字段/profile、实际消息数与规范消息序列摘要，以及模型外沙箱身份/配置摘要；确认性运行还记录 exposure root 稳定标识、claim ID、索引写入前/后代摘要与 `EXPOSURE_STARTED` 时间；
- 每项输入的 DAT-qualified immutable `data_snapshot_ref`、DAT qualification evidence 引用、snapshot digest、许可或使用限制、取得时间、`information_as_of`、规范事件时间 UTC 半开区间、缺口与修订说明；请求引用及其与资格材料的核对结果；
- 事前比较对象、数据切分、成本和现实约束，以及运行前已经固定的判定依据；
- 实际执行的方法、参数、随机性控制、搜索范围和未执行部分；
- AI 是否参与、能够核对的客户端/模型公开标签、角色、非秘密协议修订、输入摘要和实际资源/供应商成本；未知时明确 `UNKNOWN`，不保存完整聊天、模型私有状态或供应商会话作为重演输入；
- 完成、失败、证据不足或无法判断的结果及其直接原因；
- 输出文件摘要、重演方式和可删除的中间产物范围。

输入来源、摘要、时间边界或研究源码身份缺失时，该运行不得进入正向研究证据包。输入冲突无法解释时必须保留冲突，并把受影响结论标为证据不足或无法判断，不能选择有利输入继续。

## 2.4 研究证据包【ALP-RSCH-EVD-002-DEF】

研究证据包是面向经济证据判断和正式策略晋升决定的不可变文件集合。可引用运行必须来自干净 Git commit；脏工作树运行只可用于探索，不能冻结为证据，必须先提交明确源码再从干净进程重跑。证据包至少包含：

- 一个由 Pydantic 模型生成的规范 JSON 研究运行清单、源码 commit、依赖锁摘要及其摘要；源码 commit 必须从受保留 Git ref/正式历史可达，或由包同时保存规范源码快照及其摘要；
- 被评价对象、经济机制、适用与不适用范围；
- 事前基准、成本、数据切分、搜索范围和现实执行限制；
- 研究政策身份与摘要、任务预算、`confirmation_decision_family`、纳入的全部经济主张族、`confirmation_claim_type`、全部候选/参数/适应性修订计数、全局 α/FDR/FWER 预算与方法、各漏斗门结果、确认性暴露索引代摘要；确认性运行还包含 exposure root 稳定标识、不可逆 claim ID、写入前/后摘要、最终 holdout 暴露次数及适用的多重检验调整；
- 支持结果、最强反例、失败运行、无效运行和未解决未知；
- 数据泄漏、选择偏差、代理差异、样本不足与不可重演风险；
- 对正式策略实现所需输入、动作能力和产品约束的差距说明；
- 由可信父进程 runner 绑定请求/源码/锁/输入/引擎配置、harness 与 observation/intent 协议后独占生成的 NautilusTrader 原始事件/订单/成交/权益报告、`ReportProvider`、内置 statistics、规范消息序列摘要与 Plotly HTML；需要本地交接明细时使用 Parquet，不另造表格数据库；
- 包内文件清单、每个文件的 SHA-256、包完整性摘要与核心可重复性摘要。

证据包必须把“本地完整 payload”和“AI 可见审查视图”分开。完整 payload 可以包含受许可约束的行级 Parquet、逐事件 observation/intent transcript、Plotly HTML、原始运行日志和规范源码快照，只能留在本地允许根；`ai_review_view.json` 只包含有界聚合指标、不确定性、硬门结果、失败/未知、来源摘要和完整 payload 摘要，不含逐事件 transcript、行级数据、任意 HTML、自由文本日志或秘密。证伪和最终选择角色只读取该视图、允许的冻结源码与候选全量索引；只有适用 L4 明确分类为允许模型处理的数据才可额外暴露。供应商网络、遥测、保留或训练政策不因文件位于证据包而获得读取许可。

只有通过全部硬门槛、且已按预选单假设全局 α 或 strong-FWER 取得个体确认性资格的候选，才可以附带“最终候选审查材料”。只通过 BH/BY set-FDR 的完整集合只能附带集合级研究材料，任何事后排序/抽取成员都不能改名为最终候选。最终候选材料必须把经济机制、成本后样本外与确认性最终 holdout 结果、候选全量、确认性主张类型与错误率控制、参数邻域和分段稳定性、成本/滑点/延迟/缺失数据压力、最强证伪、失败与未知、实现差距、预期维护负担和证据摘要压缩为项目所有者可审查的单一入口，并允许项目所有者在本地展开完整 payload。“最终候选”只表示可进入人工审查，不表示已经形成投资 Alpha、正式经济证据判断、正式策略或上线资格。

研究证据包只冻结“研究当时交付了什么”。它不宣称研究结论当前仍适用，也不替代 ALP 的经济证据判断。数据、经济机制、成本、市场结构或正式策略语义变化后，旧包只能作为历史输入；重新引用前必须说明其仍适用的范围。删除工作分支或 worktree 前必须证明每个仍保留的正向、负向或零候选包所引源码从耐久 Git ref 可达，或已包含足以重演的规范源码快照；squash、分支删除或垃圾回收不得使证据只剩不可达 commit ID。

Pydantic 模型是研究政策、AI 研究任务、研究用例、运行请求/回执、研究运行清单、`PointInTimeObservation`、`ResearchIntent`、候选全量/确认性暴露索引、AI 审查视图和包清单 schema 的唯一源码；需要机器 schema 时从模型生成，不再同时手写第二套 `research/schemas`。规范 JSON 字节固定为无 BOM 的 UTF-8、对象键按 Unicode 码点排序、紧凑分隔符、数组保留声明顺序且拒绝 NaN/Infinity；文件路径统一为包根相对 POSIX 路径并按同一规则排序。适用 L4 必须把实际规范化实现与 Python 版本固定进研究锁和资格证据。

`bundle_integrity_digest` 是对“移除该摘要字段后的规范清单正文”与全部 payload 的有序 `{path, size, sha256}` 索引共同计算的 SHA-256，用于确认一个冻结包没有被原位改变。`core_reproducibility_digest` 只覆盖源码 commit/源码快照摘要、worker 源码摘要、依赖锁摘要、输入摘要、经济问题、可信 harness 版本、observation/intent schema 摘要、规范消息序列摘要、固定引擎配置/参数/随机性控制以及可信父进程从 canonical artifacts 重新计算的稳定规范 JSON 核心输出，用于比较独立重演；运行 ID、墙钟时间、绝对路径、日志、Plotly HTML、AI 客户端/模型标签、提示或会话身份、沙箱供应商易变元数据和组件易变元数据必须排除并在清单中声明。Plotly 只输出可离线打开的 HTML，本契约不增加 Kaleido、浏览器截图或静态图片渲染依赖；HTML、沙箱资格来源和非秘密 AI 创作来源可以进入包完整性摘要，但不得进入核心可重复性摘要。Notebook 输出、组件 cache、聊天记录和交互会话状态不得进入任一权威摘要后冒充可重演输入。

证据先写入状态为 `INCOMPLETE` 的临时目录；全部 payload 关闭、摘要复算和 schema 验证通过后，写入不参与摘要的 `COMPLETE` 标记，并在同一文件系统内原子重命名为以 `bundle_integrity_digest` 标识的最终目录。缺少 `COMPLETE`、摘要不符或仍位于临时目录的包不可引用；最终目录不得覆盖或原位修改，任何变化都产生新摘要和新目录。两个独立重演的包完整性摘要可以因明确排除的 HTML/日志而不同，但核心可重复性摘要必须相同，否则保留漂移并把结论降为证据不足或无法判断。

## 2.5 不利结果与可重建材料【ALP-RSCH-EVD-003-REQ】

会改变后续研究或晋升决定的失败、无效、证据不足、负面和冲突结果必须进入研究证据包或被其稳定引用。不得只保存最佳参数、最佳图区间或成功运行，也不得以重命名策略、改变样本起点或删除失败目录把同一经济主张伪装成新研究。

下载缓存、派生表、Notebook 临时输出、图片和可从清单重建的本地 catalog 可以删除。研究证据包仍被经济证据判断、正式策略来源追溯或后续反例使用时必须保留；全部消费者结束且对应决定不再引用时可以删除。Git 只保存实际提交过的研究源码历史，不复制未提交过程版本或建立 archive 目录。

适合仓库体量的小型冻结包可以进入 `research/evidence/`。仍被决定引用但不适合 Git 的大型包不得放在可删除的 `runs/`；适用 L4 必须先固定一个非数据库、非服务的耐久文件根，记录稳定存储标识、相对路径、`bundle_integrity_digest`、保留期、备份/恢复方式和按来源重新取得后的摘要验证。没有该位置、完整性复验和恢复证据的大型包不得作为可引用研究证据。此边界只允许文件存储，不授权制品平台、实验数据库或同步服务。

`confirmation_exposure_root` 与大型证据文件根是两个不同责任：前者很小，但在任何已暴露 `data_snapshot_ref` 的区间仍可能被研究复用时不可删除，必须固定锁/刷新/原子替换语义、索引代校验、备份/恢复和损坏演练；后者保存可引用 payload。暴露根丢失、回滚、分叉或无法证明恢复到最新代时，所有确认性运行立即禁用，不能从当前 worktree、胜者证据包或 AI 记忆猜测重建未记录的失败/崩溃 claim。

# 3. 研究方向、方法漏斗与 AI 分工【ALP-RSCH-MTH-001】

## 3.1 研究方向选择【ALP-RSCH-DIR-001-REQ】

研究方向按经济机制、现实可取得信息、Halpha 可执行性和个人资本可能具有的非对称性选择，不按 AI 新颖度、回测数量或策略标签选择。一个 AI 研究任务只能启用其数据、时间语义、成本模型和动作能力均满足的少量方向；前置条件不满足时明确等待或不支持，不用代理叙事补齐。

| 方向族 | 可证伪机制与优先原因 | 启用前置条件 |
|---|---|---|
| 状态条件下的中低频延续或反转 | 参与者反应不足、过度反应、流动性恢复或波动状态可能在非极低延迟窗口留下有限优势；可优先使用已有市场数据、低换手和少参数表达 | 事前状态可观察；信号窗口晚于数据可得时点；净成本后仍有空间；不依赖队列优先或机构基础设施 |
| 带时点的事件与流量响应 | 资金费、成交/持仓变化、波动或公开事件可能产生有方向且会衰减的响应 | 事件、修订和发布时间可追溯；不存在决定后回填；反应窗口与 Halpha 完整计划/动作链时延相容 |
| 持有收益、资金费、基差或流动性补偿 | 承担库存、期限、方向或流动性约束可能获得补偿 | 唯一正式引擎已实际建模资金费、借贷、费用、滑点和退出；缺任一决定性成本时保持 `NOT_MODELED` 或证据不足 |
| 横截面或相对价值 | 同时比较对象可减少普通市场暴露并利用相对错价 | 有无幸存偏差的当时可交易对象集、同步事实、可实现的多标的/做空/借贷能力和共同成本口径；否则不启用 |
| 用户专属上下文的稀疏机会过滤 | 用户长期积累且难以由公共信号复制的上下文可以帮助只选择少量机会 | 输入在决定前已时间戳化、可追溯并能形成确定特征；AI 叙事、记忆或聊天内容本身不能充当事实或特征来源 |

优先顺序是：已有数据与正式执行能力可覆盖、机制简单、参数少、换手与维护负担低的方向，随后才是需要新增数据或更复杂模型的方向。纯公共指标穷举、无机制黑箱搜索、依赖极低延迟/队列/专有基础设施、只有忽略成本才成立、无法恢复当时可得信息或需要机构规模才能产生价值的方向在进入运行前拒绝。机器学习、自动特征生成和在线再训练不属于默认基线；出现明确且无法由上述简单方向满足的消费者时，先修订本 L3 和适用 L4，不由 AI 自行安装框架或扩大搜索。

| 阶段 | AI 主要工作 | 确定进入条件与失败结果 |
|---|---|---|
| 方向资格 | 生成机制与方向候选，寻找最强反解释 | 只保留满足第 3.1 节和任务范围者；其余在读取结果前拒绝 |

| discovery | 用最小样本和最简单规则检查机制方向、数量级和明显不可行性 | 只用于淘汰和有限参数选择，不形成最终正向结论 |
| 时间序列 validation | 按时间顺序走步，在未参与当前 fold 规则形成的区间比较事前基准、成本后结果和统计不确定性 | 只用于自适应淘汰、有限选择和压力发现；任一硬门槛失败即淘汰，不得用其他区间替换失败区间，也不得把其 p 值表述为确认性资格 |

## 3.3 统计、经济与现实验证【ALP-RSCH-VAL-001-REQ】

研究必须先定义会改变最终判断的一个主要经济比较和少量辅助诊断。投资 Alpha 仍按 L0/ALP 的事前基准、成本和风险调整判断；“Alpha 竞争力候选”只表示一个用例同时通过以下不可替代的方面，不是收益保证或正式结论：

- 机制方向与可观察预测在未参与规则形成的数据上没有被主要反证推翻；
- NautilusTrader 唯一回测路径给出的成本后结果优于事前现金、市场或低成本替代基准，且结论不依赖未建模成本；
- 结果在时间走步、最终 holdout、参数邻域、子区间/状态和保守执行压力下仍与声明范围相容；
- 数据、完整计划链时延、动作 profile、流动性、个人资金容量和长期维护负担允许正式实现。

时间序列按自然时间顺序切分，不随机打乱；对象池、特征、归一化、缺失填补、成本校准、阈值和超参数等全部拟合态都在每个 fold 仅由当时过去数据重建，会产生回看或重叠标签跨界时使用覆盖两者的事前 gap/embargo。discovery 可以用于机制与参数选择，validation 用于内部淘汰，确认性最终 holdout 只用于冻结假设族的一次性最终检查；同一数据不能因改名同时承担多个证据角色。前向观察和真实结果可以增强、否定或使历史证据衰减，但不能被历史回测替代。

NautilusTrader 内置 statistics 与 `ReportProvider` 是策略收益、回撤、订单、成交和绩效的唯一来源。SciPy 的 `bootstrap`、`permutation_test` 和 `false_discovery_control` 只用于任务已声明且其抽样、交换性和依赖条件成立的统计问题。SciPy 不被视为原生时间序列 block-bootstrap 引擎；Halpha 只能按预登记先构造非重叠块级观测，再由 SciPy 对块重采样或置换，块间依赖仍不满足时输出证据不足。适用 L4 必须固定有效 p 值构造、检验族、块长、确认性主张类型、BH/BY set-FDR 或预选单假设/Bonferroni strong-FWER 方法、适用依赖条件和阈值；没有满足方法前提时不能换到更有利检验。回归、HAC、协整或更强时间序列模型只有出现明确消费者且 SciPy 基线不足时，才可修订本 L3、评价 statsmodels 或其他成熟组件；当前不自研这些方法。

discovery/validation 发生的全部候选、参数和适应性修改进入搜索全量与选择偏差说明，但不能靠把次数代入 FDR 重新获得确认性含义。只有在未参与生成、调参或筛选的最终 holdout 上，由预先冻结的同一 `confirmation_decision_family` 内全部经济主张族和假设产生的有效 p 值，才可执行预声明错误控制；调整覆盖该最终选择与 holdout 上全部被检验候选，不能按主张族拆开或只保留胜者。BH/BY 只控制完整拒绝集合在其声明条件下的 FDR，不能为事后选出的单个成员提供个体保证；个体可晋升必须使用 holdout 前预选单假设的全局 α，或覆盖全部可能晋升假设的 Bonferroni/已资格化 strong-FWER。若采用分层检验或分配 α/FDR/FWER 预算，层级、预算、主张类型和合并规则必须在任何 holdout 暴露前固定并完成资格化。统计错误控制不替代经济机制、现实成本、可重演性或人工最终判断。默认只允许预登记的有限网格和参数邻域压力，不引入黑箱优化器；若结果随微小参数变化翻转，候选不得因最佳点漂亮而晋级。

## 3.4 AI 角色与可见性【ALP-RSCH-AIR-001-REQ】

| 证伪审查 | 冻结用例、全部正负运行的 AI 审查视图、允许的冻结源码和公开限制；只写审查材料 | 寻找泄漏、替代解释、压力缺口和选择性呈现；不得读取本地完整 payload、修改用例或删除运行 |

不同角色可以由同一个外部 AI 客户端或模型在新会话中执行；角色分离用于减少上下文污染和越权，不声称模型独立。AI 只能引用机器回执、文件摘要和明确材料形成理由，无法追溯的叙事标为不支持。AI 编写源码的静态规则检查、导入 allowlist 和证伪审查只是防错与研究资格门，不是安全边界；实际执行仍必须服从第 1.2 节的模型外沙箱。项目所有者是最终候选的唯一必要人工审查者；未形成最终候选时不要求人工为漏斗补一个结果。

任何完整数据或 holdout 运行中，AI 编写的 study 都只能作为模型外沙箱内的短生命周期 worker；可信父进程不得 import、`exec` 或在同一解释器调用它。父进程逐事件发送 schema 校验的 `PointInTimeObservation`，只含该时点已经可得的规范事实、事前允许的指标与当前模拟状态，不含数据/holdout 文件路径、未来标签、未来事件、canonical writer、确认性计数或产品对象；worker 只能返回与同一 sequence 对应、枚举 profile 和有界数值参数组成的 `ResearchIntent`。`ResearchIntent` 只是研究仿真提案，不是 `ExecutionAction`、订单事实、交易计划、权限、资本决定或当前动作；只有可信父进程按固定 harness 将其映射为 NautilusTrader 仿真订单。

worker 可以在一次运行内维护由其源码和已收到 observation 决定的纯研究状态，但不能持有引擎、组合、时钟、数据切片、指标计算器、报告器或 artifact writer。完整 observation/intent transcript 只留在模型不可读的本地 payload，AI 只取得聚合审查视图与机器回执。若某个策略需要任意引擎回调、数据文件访问、动态代码、自由文本命令或不能由该有界协议表达，它不能绕过边界进入确认性证据；必须缩小方向，或先修订本 L3/ENG-003 并重新评估复杂度与安全性。

## 4.4 最小 Halpha 自有代码与 CLI【ALP-RSCH-DEP-004-REQ】

本契约只保留三项研究工作区自有薄责任和一个 CLI；适用 L4 固定实际文件，初始可以分别落在以下单文件中。若单文件妨碍审查或测试，可以在同一责任内部拆分普通 Python 文件，但不得借拆分增加框架包装层、插件系统、服务或第二协议：

- `data_contract.py`：声明 Pandera 输入 schema，只解析政策允许的 DAT-qualified immutable `data_snapshot_ref`，核对 DAT qualification evidence、snapshot digest 与规范 UTC 半开区间，记录引用和核对结果，并把已通过检查的数据直接交给 `BarDataWrangler` 与 `ParquetDataCatalog`；

研究用例本身位于各自 `studies` 源码中，入口只能作为读取规范 stdin frame、返回一个规范 `ResearchIntent` 的短生命周期 worker 表达冻结规则；可信父进程从不导入该入口。worker 可使用任务允许的 Python/NumPy 纯计算，但不能导入或取得 NautilusTrader、runner、harness、报告器、artifact writer、数据文件或产品模块。AI/study 不能写入或提供 `PASS`、p 值、调整后 p 值、候选计数、成本后绩效或最终资格；runner、harness 和 gate 属于共享研究协议的串行源码，不由单个研究活动修改。源码静态扫描必须拒绝未声明文件/环境/网络/进程 API 和非允许导入，但不能替代模型外执行沙箱。

`run` 的可信父进程在 worker 沙箱之外持有数据文件、时钟、NautilusTrader 引擎、原生指标、组合/订单/成本状态、确认性计数和 canonical writer；它为每次运行启动一个新沙箱 worker，通过有界 framed stdio 逐事件发送 `PointInTimeObservation` 并接收 `ResearchIntent`。worker stdout 只能承载与 sequence 一一对应的协议 frame，stderr 由 launcher 捕获到模型不可读的本地 payload；额外、未分帧、重复、乱序、超大、超时、非有限数值、未知字段/profile 或越过参数范围的响应都使运行失败。父进程验证 intent 后才由固定 harness 映射为 NautilusTrader 仿真订单；worker 不取得数据文件、未来事件、父进程内存、引擎引用或输出写路径。

当请求阶段为 holdout 时，`run` 还是 `confirmation_exposure_root` 的唯一运行时写入主体。普通 `validate` 和各 worktree 的 Git 索引只可预检；父进程必须按第 2.2 节先解析政策允许的 `data_snapshot_ref`，核对 DAT qualification evidence、snapshot digest 与规范 UTC 半开区间，再在打开 holdout 前取得锁并耐久写入不可逆 claim，随后把 `confirmation_exposure_key`、`data_snapshot_ref`、资格证据引用、snapshot digest、规范 UTC 半开区间、claim ID 与索引写前/写后摘要绑定到 canonical artifacts。请求不得提供或覆盖 exposure root/锁路径，也不得创建、改写或用路径、名称、digest 代替政策允许的引用；worker 与 AI 角色不可见该根。父进程或宿主在 claim 后任何失败都返回失败回执但不撤销暴露；`gate` 继续只读 claim 与 artifacts，不拥有补登记、删除或修复历史暴露的入口。

`run` 只由可信父进程生成引擎原始报告、规范消息序列摘要与只读 canonical artifacts 后退出；`gate` 从新的干净可信进程读取 canonical artifacts、冻结清单、全部运行回执和确认性暴露索引，重新计算 SciPy 统计、候选计数、成本和硬门，忽略 worker 自行写入的指标、状态和报告。伪造 `PASS`、替换报告、删除候选、修改计数或试图污染父进程必须因进程/能力边界、schema 或来源/摘要不匹配而失败。三个责任不得接管组件缓存、事件循环、指标、订单模型、撮合、PnL、绩效、Notebook、图表引擎、依赖解析或平台连接；某项薄代码长期只有转发作用或组件已有等价公开能力时删除，不为“未来可替换”增加抽象。

- 经济机制、适用范围、主要反例和失效条件仍然明确；
- 数据来源、时间切分、搜索范围、成本和现实约束可追溯；

# 8. 成熟本地组件与文件交接的选择依据【ALP-RSCH-CMP-001-RAT】

Halpha 的长期目标需要持续发现、证伪并晋升策略，但不需要拥有通用研究或 AI 平台。复用正式产品已经选择的 NautilusTrader 可直接减少 live/backtest 语义漂移；pandas/PyArrow、Pandera、SciPy、Plotly、Pydantic 和 Jupyter 生态分别承担成熟的窄责任，使 Halpha 只维护与经济问题、预登记、来源、cutoff、搜索全量、硬门、失败保留和晋升有关的薄代码。

外部 AI 直接调用人也能调用的 JSON/CLI 契约，比在 Halpha 内集成模型 SDK、agent framework、MCP、向量库或会话状态少供应商认证、工具循环、重试、追踪、成本和升级责任。研究的竞争力来自更好的问题、严格预登记、快速证伪、完整搜索记录、未暴露 holdout 和现实执行检查，不来自代理数量。模型不可用时只保留对固定请求的恢复性人工重演，因而 AI 提高吞吐但不成为证据或运行依赖。

允许 AI 自主执行任意 Python 而只依赖提示词、schema、venv、AST 扫描或普通 `subprocess`，不能支持“人工仅最终审查”。最低可接受方案是优先复用外部 AI 客户端或宿主已经具备、且能通过恶意 fixture 的成熟模型外沙箱；Halpha 只记录配置摘要和使用同一 CLI，不自研 Python sandbox。若现有能力不能证明 deny-by-default 文件/网络/进程/秘密/资源隔离，就保持完整数据与确认性研究未启用；以后比较按需 OS sandbox/container 时，若引入新的镜像、依赖或构建身份，必须先修订并接受 HALPHA-ENG-003，不能由 L4 自行扩张。不能因为研究价值预期而先引入常驻平台或虚构隔离成功。

模型外沙箱也不能阻止同一解释器内的 monkeypatch，因此可信父进程与未受信 worker 的能力分离不是可选加固。observation/intent 是 Halpha 自有的窄研究策略协议，其成本不只两个 Pydantic 模型：每个新字段、状态、profile 与 intent—订单映射都会增加研究—正式实现差距和共享串行维护面。最低方案因此只为一个明确中低频方向族资格化最少字段/profile，把字段与 intent 类型数量、目标事件规模下消息率、增量墙钟/CPU/内存，以及与等价直接 NautilusTrader harness 的事件时序、仿真订单、成交、成本、PnL 和报告一致性设为硬门；数据推进、指标、撮合、成本、报告、统计与证据写入仍全部复用 NautilusTrader 和已有薄责任，不建设通用策略 DSL、服务、消息总线或持久 worker。字段/profile 持续扩张、性能越界、语义不等价，或策略表达必须依赖任意引擎回调、共享内存或动态插件时，本方案的低复杂度结论失效，应先收缩方向或修订本 L3，而不是绕过边界。

文件证据包比研究 API、实验数据库或自动同步平台少一个常驻进程、数据库、认证面、迁移线和恢复责任，又能通过源码、锁、输入与 SHA-256 重演。研究与产品的变化频率和依赖冲突由独立环境隔离，正式策略仍通过一次人工最终审查和产品内重验进入唯一运行源码。只有实际研究吞吐、多个 AI 客户端的 CLI 重复适配或手工交接错误证明文件方式已成为可测瓶颈，且新增能力的全生命周期成本更低时，才重审这一取舍；首选仍是从同一 Pydantic 模型生成的无状态本地适配，不先建网络服务。

# 9. 复杂度与适用 L4 选择边界【ALP-RSCH-CPLX-001】

本契约只增加文件型研究政策、AI 研究任务、研究用例/请求与回执、运行清单、搜索全量/确认性暴露索引、一个单宿主小型耐久 exposure control root、AI 审查视图、角色材料和研究证据包；它们由同一 Pydantic 模型族、三项薄责任、一个 CLI、一个固定可信 harness 和每次运行一个短生命周期沙箱 worker 承担，不形成产品对象、数据库或工作流引擎。ALP 拥有 exposure/observation/intent 的研究语义与失败后果；HALPHA-ENG-003 拥有锁、文件原子性、沙箱和父/worker 隔离方法，适用 SYS 提供通用运行约束，适用 L4 只记录精确选择与证据。确认性准入的短排他锁不会串行化 discovery/validation，也不要求把互不冲突的已登记 holdout 全程串行。优先复用已存在且资格通过的外部工具沙箱；没有合格能力时功能保持禁用，不由 Halpha 自研安全沙箱。它不增加交易产品业务模块、产品运行进程、数据库产品、持久业务记录族、持久工作器、模型 SDK、真实写链、授权路径或独立发布组。

适用 L4 只有在记录真实研究消费者、研究政策允许的初始单一方向族与数据/方法/门槛/预算范围、允许消费的 DAT-qualified immutable `data_snapshot_ref` 及其 qualification evidence、snapshot digest 和规范 UTC 半开区间、确认性决策族/经济主张族映射、`confirmation_claim_type` 与全局 α/FDR/FWER 预算、资格化或试点任务的精确切分与阈值、单一目标宿主和 `confirmation_exposure_root` 稳定标识/解析路径/唯一写入身份/排他锁/索引代 schema/摘要链/刷新与原子替换/备份恢复/损坏和崩溃演练、可信 `ResearchStrategyHarness` 版本、observation/intent 的精确 schema/摘要/允许字段与 profile/framing/顺序/大小/超时预算、字段/profile/intent 类型数量上限、目标事件规模与消息率、IPC 增量墙钟/CPU/内存上限、等价直接 NautilusTrader harness 的时序/订单/成交/成本/PnL/报告黄金比较、父进程与 worker 的 SID/ACL/挂载/read-write roots、AI 客户端与角色可见性、供应商遥测/保留/训练政策和允许上传数据类别、模型外沙箱身份/配置、组件版本和 hash lock，确认许可证与目标平台/Python 资格，证明 CLI 重演、未受信源码、进程内污染反例、holdout、完整 payload 与资源隔离，安排删除条件并证明不会阻塞交易产品后，才可以启用研究工作区。L4 必须保持研究常驻进程、模型调用服务、数据库、持久 worker、产品业务记录族、真实写链、授权路径和独立发布组增量均为零，并把本契约未采用组件记录为明确非选择。政策内后续动态任务由任务清单和证据包记录，不把 L4 变成 campaign 数据库。

若未来需要自动判定多个数据源、修订、重导出或切片之间的等价关系，必须先由 DAT 形成新的或修订既有 L3，定义来源身份、时间、质量与追溯契约；ALP 继续只消费 DAT 输出，不得在本契约或 L4 中自行增加 lineage 算法。

若实际实现需要 Halpha 调用模型 API、跨会话持久 agent 编排、第三个产品持久进程、第二产品数据库、第二正式量化栈、独立产品发布、产品研究界面、远程入口或自动平台同步，本契约的复杂度结论失效，必须停止实施并由相应语义所有者修订并接受对应设计。
