# Halpha 单场所交易环境事实能力契约

**文档编号：** HALPHA-DAT-002  
**层级：** L3  
**L3 类型：** DOMAIN  
**主要语义所有者：** DAT  
**语言版本：** zh-CN  
**上位文档或条款：** HALPHA-DAT-001  
**直接依赖：** 无；DAT 只生产事实，不反向依赖消费者决定  
**直接消费者：** HALPHA-ALP-002、HALPHA-TRADEPLAN-002、HALPHA-CAP-002、HALPHA-EXE-002、HALPHA-OUT-002、HALPHA-UX-002、HALPHA-SYS-002  
**适用纵向约束：** HALPHA-SYS-001、HALPHA-ENG-001  
**本文档负责：** 单场所交易所模拟盘与真实资金环境实际使用的闭合 K 线、标记价格、最优报价、工具规则、账户、订单、成交、手续费、资金费和持仓事实；事实来源、环境、时间、截止点、摘要、ExecutionAction/激活归属、缺失结果、追加与恢复契约  
**本文档不负责：** 形成信号、条件结论、资金授权、动作决定、复盘评价；保存全量行情或框架内部状态；规定当前场所、账户、工具、采样、阈值、保留期、NautilusTrader 精确版本或适配器配置  

---

# 0. 设计结论【DAT-RT-SUM-001】

DAT 只保留一个追加式 `VenueFact` 记录族。NautilusTrader 负责订阅、历史读取、K 线、工具模型、订单事件和连接恢复；Halpha 只保存计划判断、资金检查、动作、保护、责任闭合或复盘实际引用的事实。未被使用的行情留在组件内，不建设行情仓库、事实工作流或第二状态源。

---

# 1. `VenueFact` 记录族【DAT-RT-FACT-001】

## 1.1 当前需要的事实【DAT-RT-FACT-001-REQ】

`VenueFact.kind` 只支持以下有直接消费者的类型：

| kind | 最小业务载荷 | 主要用途 |
|---|---|---|
| `CLOSED_BAR` | bar type、区间、OHLCV、`ts_event`、闭合标志 | 代码策略和计划条件 |
| `MARK_PRICE` | 标记价格、事件时间 | 未实现盈亏、增险前估值和风险减少 |
| `TOP_OF_BOOK` | 同一更新的买一价量、卖一价量、事件时间 | 动作量化和提交前检查 |
| `INSTRUMENT_RULES` | 工具状态、价格/数量步长、最小数量与名义值、精度 | CAP 与 OrderFactory 输入 |
| `ACCOUNT_STATE` | 余额、账户/持仓模式、当前工具的保证金模式、杠杆和费率，以及同一查询截止点 | 激活和提交前检查 |
| `ORDER_STATE` | 普通或条件订单身份、客户端身份、条款、状态和累计成交量 | 动作结果、保护和闭环 |
| `FILL` | 成交、订单与客户端身份、价格、数量、已实现盈亏和时间 | 持仓、最大损失和复盘 |
| `COMMISSION` | 实际手续费金额、资产、关联成交/订单和时间 | 本激活最大损失与复盘 |
| `FUNDING` | 实际资金费收入或支出、资产、工具、结算时间 | 本激活最大损失与复盘 |
| `POSITION_STATE` | 工具、方向、数量、入场/标记价格、未实现盈亏和查询截止点 | 风险减少与责任闭合 |

预计值和组件内部估值不能替代已发生的费用、资金费、订单、成交或持仓事实。资本数值使用 Decimal，并保存币种、单位和来源精度；缺失、过期或工具不匹配时不得新增风险。

## 1.2 最小字段【DAT-RT-ID-001-REQ】

每条 `VenueFact` 至少包含：

| 字段 | 约束 |
|---|---|
| `venue_fact_id` | 由环境、事实种类、来源对象身份和内容摘要稳定派生 |
| `environment_id`、`venue_ref` | 隔离模拟与真实环境；不得以显示名代替稳定身份 |
| `account_ref`、`instrument_ref` | 按 kind 必填；不适用为空，不造默认值 |
| `source`、`source_object_id` | 查询、推送或明确派生方法，以及场所或组件公开身份 |
| `source_time`、`received_at`、`cutoff` | 来源时间、本机取得时间和本次查询截止点；不互相回填 |
| `content_digest` | 规范化内容摘要；本地重试时间不进入摘要 |
| `payload` | 对应 kind 的 Decimal 业务值和公开身份；不保存秘密 |
| `activation_ref`、`action_ref` | 只有稳定外部身份能够精确对应同环境激活或 `ExecutionAction` 时才填写 |
| `supersedes_ref`、`change_reason` | 仅在来源修正先前事实时填写；原事实不覆盖 |

同一来源对象和内容返回同一事实；同一来源身份出现不同内容时追加并标明变化，消费者在冲突消除前按 `UNKNOWN` 处理。场所或组件派生值必须说明方法和输入，不能伪装为场所直接报告。

# 2. 来源与归属【DAT-RT-ATTR-001】

场所查询和带稳定身份的场所推送可以说明场所当时报告的状态。组件缓存、组合、风控投影和合成订单事件只用于寻找或比较事实，不能单独证明场所已受理、实际费用、资金费、持仓归零或责任闭合。

Halpha 订单、成交和费用只有在客户端身份能够精确映射到同环境已持久 `ExecutionAction`，且账户、工具和条款一致时，才填写动作与激活引用。资金费只有在场所流水和该时间范围能够唯一归属时才计入激活；否则保持外部或未知。

用户接管后的查询仍可证明账户持仓为零或订单已经清理，但用户在官方入口完成的活动不改写为 Halpha 动作，也不能补造精确激活收益。身份不完整、反向或新增风险的外部活动阻止归属和责任闭合。

Demo 与 Live 事实只能引用各自环境的账户、激活和动作。跨环境引用、复制或把 Demo 结果提升为 Live 事实一律拒绝。

# 3. 消费、未知与修正【DAT-RT-USE-001】

消费者按用途、环境、账户、工具、事实种类和截止点查询。DAT 直接返回满足要求的当前事实引用和值，或者返回 `UNKNOWN` 及具体原因；查询结果不是持久对象。消费者形成计划或动作决定时，只把实际使用的事实引用、截止点、输入摘要和必要的未知原因写入自己的 `PlanActivation`、`PlanEvent` 或 `ExecutionAction`。

关键事实缺失、过期、来源冲突、时钟不可信、连续性无法证明或归属不明时，不得沿用最近值、填零、拼接不同截止点或使用替代值放行。受影响的增险停止；可靠查询、保护、退出和用户接管继续。

闭合 K 线只保存组件明确标为闭合的 bar；最优报价的价量必须来自同一更新。错过窗口后取得的新事实只支持新的当前判断、风险减少、核对或复盘，不补发历史增险。

来源后来修正事实时，只在 `VenueFact` 中追加一条引用原事实和原因的新记录。引用原事实的领域按自己的当前责任决定是否追加新判断；DAT 不建立修正任务、传播 worker 或追溯生成动作。

# 4. 组件与恢复【DAT-RT-NT-001】

NautilusTrader Binance 适配器承担行情与账户连接、订阅、历史请求、K 线、工具/数值类型、公开事件和连接恢复。HalphaCoordinator 从公开输出中挑选实际使用事实并写入 `VenueFact`；不建立 DAT 摄取工作器、持久检查点或第二重连状态机。

组件提供的默认费率和内部回退只是技术输入；只有场所成交、账户流水或经直接验证的只读查询才能形成实际费用事实。

若公开事件缺少订单结果、实际费用、资金费或账户流水，只允许在同一 Executor 内复用 NautilusTrader 同一连接栈增加窄只读查询；它不得包含写方法、建立第二客户端、缓存、数据库或恢复循环。仍无法取得时保持未知，不引入第二 SDK 或状态源。

重启后，各消费者按未闭合责任重新查询所需事实。DAT 不恢复全量行情历史，也不拥有计划或动作；后来取得的事实不能清除已经锁存的停止新增风险或补发错过动作。是否继续原激活、结束责任或创建新激活仍由 TRADEPLAN 与 EXE 根据当前事实决定。

# 5. 验证契约【DAT-RT-TST-001-REQ】

实现至少证明：

1. 当前十种事实保存稳定来源、时间、截止点、Decimal 和摘要；重复输入幂等，变化与修正追加而不覆盖；
2. 未闭合 K 线、过期或异常报价、缺失价格和时钟冲突保持 `UNKNOWN`，不使用最近值、零或替代值；
3. 订单、成交、费用、资金费和持仓只从经验证的公开输出或唯一窄只读查询形成；预测值不能冒充实际值；
4. Halpha 事实只有精确匹配同环境 `ExecutionAction` 时才归属；用户接管和其他外部活动不冒充 Halpha 动作或确定收益；
5. 重启只按开放责任补取事实，不补发历史增险；只读补充没有写方法、第二客户端、数据库或恢复循环；
6. Demo 与 Live、不同账户和工具的同名对象不能合并或交叉引用。

# 6. 明确不建设与 L4 边界【DAT-RT-NON-001-REQ】

不建设全量行情仓库、本地订单簿、通用指标仓库、自研协议泵、第二账户账本、事实工作流、数据服务进程或与 NautilusTrader 平行的行情/账户适配器。

L4 只固定当前依赖和适配器版本、实际场所/账户/工具、订阅与查询配置、允许用途、时效与 cutoff、只读补充选择、保留期、平台验证结果和验收证据；本 L3 不声称这些当前已经可用。
