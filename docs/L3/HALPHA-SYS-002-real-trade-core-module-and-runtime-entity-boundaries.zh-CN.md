## 1. 模块与所有权

产品按语义分为 planning、capital、venue integration、outcomes 和 user workbench 等模块。每项业务记录只由拥有模块持久化；其他模块调用公开领域能力或读取已提交结果。

模块边界是代码和状态所有权，不要求一一对应进程、服务或数据库。当前模块和物理记录映射由代码与 L4 记录，不作为长期数量上限。

## 2. 运行角色

产品保持交互应用和隔离执行两个逻辑角色。交互角色提供 Web、API、认证、领域请求和查询投影，不创建交易节点或持有真实账户交易凭据。执行角色运行量化组件、发现已启用计划和未决动作，并且是唯一可以发起交易所变更请求的角色。

两个角色通过同一环境的已提交产品记录协作。进程内通知只降低延迟，丢失后能从数据库重新发现；不使用内存队列或同步 RPC 承载唯一业务责任。

角色当前映射为多少进程、实例和操作系统任务属于 L4。只要交互角色不能取得真实账户交易凭据或发起交易所变更请求，未来可以按实际问题调整映射。

## 3. 环境与存储

Demo 和 Live 复用领域代码，但绑定不同环境、账户、端点、凭据引用、动作和事实身份。权威存储使用独立数据库或明确隔离的 schema；具体方式由当前配置决定，不能跨环境引用或默认回退。

只读运行不取得产品写 repository、私有账户、执行客户端或交易秘密，其输出可删除且不能转换为产品事实或动作。

## 4. 唯一写边界

执行角色只有在 EXE 动作已持久化、CAP 检查当前且环境与条款匹配时，才能调用场所客户端。系统在实际传播范围内保持一个写执行者；互斥的当前实现方式由 L4 记录。

量化组件承担行情、时钟、策略运行、订单对象和公开连接恢复；Halpha 不复制其通用运行能力或建立第二写适配。

## 5. 启动、停止和恢复

启动或非计划恢复按以下顺序：

1. 取得当前运行身份，并确认当前环境只有一个交易所请求执行者；
2. 验证环境、数据库、迁移和非秘密配置；
3. 连接只读场所能力并取得当前账户、订单和持仓事实；
4. 让 TRADEPLAN、CAP、EXE 和 DAT 分别检查自己拥有的未闭合状态；
5. 只有所有结果一致且不会补发错过动作时，按固定决策依据恢复尚未闭合责任：`STRATEGY_SIGNAL` 重建策略运行，`DIRECT_EXECUTION` 直接从已保存激活、编译快照和事件身份继续，不创建虚构策略。

任一条件未知或冲突时，系统停止受影响新增风险和新的交易所变更请求，只保留查询、已有保护、退出和用户接管。具体未知请求由拥有该动作的领域记录，SYS 不建立恢复状态机。

正常停止先阻断新动作形成和交易所变更请求，再保存未闭合责任并停止运行组件。数据库不可用时，不以内存继续形成业务记录或发起交易所变更请求。

所有承担产品持续责任或绑定产品监听端口的 Halpha 长期进程都必须进入同一个生命周期控制入口。该入口联合检查操作系统任务、当前仓库及已登记 Git 工作树中的产品进程树和 TCP 监听；能确认属于项目但不能归入受管任务或有效外部登记的长期进程或监听必须显式标为未登记服务，不得静默遗漏。短时构建、测试和迁移命令不因此变成长驻服务。

独立服务不得被产品运行代码按名称、路径、模块或命令行特征引用、导入、启动或停止。独立服务如需进入统一状态视图，只能由服务生产者通过通用外部登记协议主动提交服务标识、PID 和本地 TCP 监听；生命周期入口用操作系统监听实况交叉核验，登记失效或所有权不匹配时显式告警。有效登记只授予状态可见性，不授予产品启停权，也不把独立服务变成产品运行依赖；统一停止必须跳过这些服务。

从命令行特征识别出产品服务名称，只能用于发现未登记产品实例，不能证明该进程由生命周期入口持久管理。没有可核验的操作系统任务时，即使产品服务名称已知且可停止，也必须显示为未登记和仅发现，不能标记为完整受管。

生命周期控制入口主动生成的命令帮助、表头、状态、结果标签和错误前缀只使用 ASCII 英文，避免依赖 Windows 终端的本地编码。路径、配置值、操作系统错误、外部响应或日志等被动透传内容保留来源语言和编码，不要求翻译，也不改变其事实含义。

产品长期进程只能由生命周期控制入口或该入口管理的操作系统任务启动。产品组合启动只包含产品运行角色，不自动启动备份作业或独立服务。停止任务管理的服务时先禁用其自动重启来源，再请求受保护的正常停止；超时默认失败关闭，只有显式强制参数才能转为操作系统强停。停止未登记服务前必须重新确认目标进程仍属于已登记工作树，不能仅凭曾经观察到的 PID 操作；外部主动登记服务由其自身操作入口停止。

进程已经停止只证明对应运行实体不再执行，不证明订单、持仓、保护或其他业务责任已经闭合。需要继续原责任、退出或用户接管时，仍按本节恢复与接管契约判断。

## 6. 复杂度与验证

新增模块、进程、数据库、持久工作或第二操作入口必须由实际并发、隔离、故障或维护问题证明。当前数量不是稳定验收标准。

至少验证状态所有权、数据库事务提交后才发起交易所变更请求、交互角色没有交易所凭据、环境隔离、重复与恢复不产生第二动作、唯一 Executor 丢失时禁止新的交易所变更请求、项目长期进程和监听不存在未报告的管控外实例，以及停止和接管仍可达。
