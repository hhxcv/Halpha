# Halpha 个人交易工作台交互设计

**文档编号：** HALPHA-UX-002  
**层级：** L3  
**L3 类型：** DOMAIN  
**主要语义所有者：** UX  
**所属实现模块：** Web 前端与应用 API  
**语言版本：** zh-CN  
**上位文档或条款：** HALPHA-UX-001  
**直接依赖：** HALPHA-TRADEPLAN-002、HALPHA-DAT-002、HALPHA-CAP-002、HALPHA-EXE-002、HALPHA-OUT-002  
**直接消费者：** Web 产品实现  
**适用纵向约束：** HALPHA-ENG-001、HALPHA-SYS-001  
**本文档负责：** 工作台的信息组织、状态表达、命令反馈、安全交互和视觉底线  
**本文档不负责：** 领域状态机、资金计算、交易所变更请求、当前页面数量或精确组件版本  

## 1. 设计结论

工作台服务于单一所有者完成计划、激活、观察、控制、核对和复盘。界面应直接显示会改变决定的信息，不建立通用任务、审批、回执或通知平台。

当前路由、页面数量和主题参数属于 L4 或实现，不在本文件固化。长期信息结构按用户任务划分为计划、当前激活与操作、账户与环境三个区域；页面和任务较少时可以合并呈现。

## 2. 共同交互原则

- 服务端事实是权威来源，浏览器缓存不得推进领域状态；
- 状态改变必须显示明确结果和最新服务端版本，不做乐观成功；
- 请求超时或结果未知时，按原请求身份查询，不重新提交；
- Demo、公共行情只读和真实账户交易环境始终清晰可辨；
- 金额、数量、价格和比率以精确十进制文本输入和显示；
- 服务端与持久事实保留 UTC 时间，所有用户可见时间统一转换为 `Asia/Shanghai`（UTC+8）并带时区标识；
- 未知、过期、冲突、部分成交和保护缺口必须直接可见；
- 首屏面积与信息对当前决定的价值相称；低价值标题、背景说明、空状态解释、重复身份和技术元数据不得以大块容器、大段留白或重复声明占据主要区域，宜压缩为短标签、单行状态或按需展开；
- 可增长列表默认只显示最近的有限条目，并通过显示更多或分页继续读取；
- 高影响操作要求用户看到对象、范围和后果，但不叠加无业务意义的多层确认。

命令结果统一表达为 `APPLIED | REJECTED | UNKNOWN`，并提供原因、当前版本和可执行的下一步。

信息载体采用以下最小选择：

- 需要同时比较、共同决定或连续操作的当前内容放在同一页面；
- 同一任务内互斥的视角、状态范围或当前与历史内容使用页签，不在一个滚动流中上下堆叠；
- 当前对象的解释、参数和次要证据使用局部展开、抽屉或浮层，在不离开任务的情况下查看；
- 弹窗只用于必须打断当前操作的后果确认或短时决定，不承载普通浏览和长篇说明；
- hint 或 tooltip 只解释短术语、单位和控件，不承载风险、未知、失败或唯一事实；
- 需要稳定地址、独立恢复、跨入口访问或大量深层证据时才使用独立页面。

工作台采用一套固定的亮色视觉体系，不提供多主题切换：

- 浅灰操作轨承载品牌、环境和主导航，近白顶栏承载当前页面上下文，近白内容面承载交易事实与操作；三者保持稳定位置和清楚边界；
- 文字按决策价值使用主文字、次要文字和弱化文字三级中性色：结论、关键状态和主要数值使用主文字，说明与上下文使用次要文字，标签、时间和技术元数据使用弱化文字；不得把全部文字处理成同一黑色、字号和字重；
- 暖黄色是结构强调色，只用于当前导航、当前页签、区域内主要行动和键盘焦点；需要文字形式的次级操作时使用与暖黄色同属当前视觉体系且满足对比度的深色强调文字，不铺满普通面板；
- 成功、警告、危险和信息分别使用固定的绿色、黄色、红色和蓝色语义色族，不随结构强调色变化。每个语义色族统一拥有图标或边界色、浅背景和可读文字色；普通提示优先使用浅背景或中性表面配语义边界，只有必须立即识别的危险状态才使用大面积实色，并始终配合文字、图标或原因；
- 页面标题、区域标题、关键状态或数值、正文、标签与时间形成递减且稳定的字号和字重层级；强调色和大字号只提升当前判断所需内容，不用于普通说明或技术元数据；
- 买卖方向和盈亏颜色不得替代业务标签；
- 交易方向和盈亏使用红绿配色辅助快速辨认，默认红跌绿涨；用户可在设置中一键切换为红涨绿跌，选择在当前浏览器保存并作用于全部工作台页面。做多、做空和盈亏始终同时保留文字或正负号，零值与未知值不着涨跌色；
- 视觉层级优先由表面色差、间距、对齐、字号和字重建立；同一选择或层级只使用一个主要强调信号，填充色已经表达选择时不再叠加常驻边框；
- 边框只用于输入、数据比较、弹窗和确需界定范围的结构或操作，不使用装饰性整行分割线，不让嵌套、相邻或并列边框重合并重复表达同一边界；键盘焦点轮廓作为临时交互反馈独立保留；
- 常规面板不使用重阴影，不使用渐变、发光、玻璃效果、装饰色块或卡片套卡片；
- 页面标题、页签、按钮、输入、列表行和事实网格保持紧凑尺度与统一几何；金额、价格、比率、时间和标识使用等宽数字，便于纵向比较；
- 每个可操作控件具有默认、悬停、按下、焦点、选中、处理中、禁用和结果反馈中实际适用的状态；按下反馈轻微且即时，动效只帮助识别状态变化并尊重减少动效偏好；
- 桌面保持稳定操作轨和工作区，窄屏将主导航移入临时抽屉并按原任务顺序单列化；优先减少列数和次要说明，不缩小关键事实或隐藏停止、退出和接管。

## 3. 计划与激活

用户应能够编辑计划草稿、查看校验问题、固定版本、预览激活范围并明确激活。固定后内容只读；修改时创建新版本。

选择或查看策略时，在该策略的选择项、计划或其他使用位置按需显示策略清单提供的价值逻辑、适用场景和实际执行行为，使策略增多后仍可理解当前选择；不在计划列表顶部建立脱离使用上下文的策略介绍区，页面也不另写一套策略规则。

当前可操作计划与历史计划使用同一计划任务中的独立页签；历史内容不占用当前计划的主扫描区域。

激活预览至少显示环境、账户、工具、方向、策略配置的交易金额、允许动作、有效期、保护与退出规则及已知限制。激活按钮只让该固定计划开始运行，不表示已成交，也不立即向交易所提交订单或其他变更请求。

工作台不提供独立资金授权、账户额度或资金分配页面。真实账户交易计划的激活可以要求一次明确确认，但不再要求额外资金授权对象或逐动作审批。

## 4. 当前激活与操作

当前激活区域应优先显示：

- 生命周期、是否允许新增风险和最近事实时间；
- 持仓、开放订单、待定动作及未知结果；
- 保护是否存在、是否完整以及缺口；
- 当前损失与激活边界的关系；
- 最近关键事件、核对结果和复盘入口。

总览以当前仓位为默认内容，至少显示交易对、方向、数量、入场价、可确认的当前价格与盈亏、保护、管理该仓位的策略和事实刷新时间；当前权威事实没有提供的强平价、保证金率等字段保持未知，不由页面推算。最近交易结果放在独立页签，不与当前仓位混排。

交易记录列表至少显示交易对、方向、时间、交易金额和净盈亏。交易详情在权威事实存在时继续显示管理策略、操作理由与触发来源、逐笔价格和数量、手续费、挂单或吃单以及持仓周期；缺失项明确显示未知，不从订单状态、计划或近似时间猜测。

停止新增风险、退出策略和用户接管必须并列可见，并清楚说明不同后果。停止新增风险不能误阻保护和减仓；用户接管后界面转为只读核对，并提供场所官方接管指引。

## 5. 账户、环境与故障

账户与环境区域显示当前环境、连接状态、事实新鲜度、凭据是否可用、进程状态和恢复结果，但不得显示秘密值。

当前环境和账户由稳定的全局载体持续标识时，任务区域不重复铺陈相同身份；只有它会改变当前操作、解释失败或提示上下文冲突时才局部重申。Demo 与 Live 一次只呈现当前环境的数据；页面外壳与局部事实的环境或账户不一致时拒绝混合展示，不以同时显示两套状态解释冲突。

后端、数据库、场所或前端局部不可用时，工作台应保留能够安全呈现的只读操作信息。无法证明当前性时，禁用增险操作并显示原因，不以空白页面或笼统错误代替。

## 6. 安全与可访问性

状态改变要求已认证会话、来源校验和防跨站请求保护。密码与秘密不进入 URL、浏览器持久存储、页面日志或错误详情。

界面以桌面为主并支持窄屏阅读。关键控件必须有文字标签、键盘焦点、错误关联、足够对比度和状态播报；颜色不能单独承担含义。避免闪烁行情、装饰动画和游戏化反馈。

## 7. 浏览器验证

根据改动影响验证真实浏览器行为，至少覆盖相关的：

- 计划校验、固定、版本只读和激活预览；
- 重复点击、超时、刷新、陈旧页面和未知结果；
- 环境隔离、保护缺口、部分成交和用户接管；
- 停止、退出与接管的不同结果；
- 未认证、会话到期、来源校验和跨站请求反例；
- 桌面、窄屏、键盘、焦点、对比度与状态播报。

浏览器检查证明交互行为，不代替数据库、场所或策略的直接验证。
