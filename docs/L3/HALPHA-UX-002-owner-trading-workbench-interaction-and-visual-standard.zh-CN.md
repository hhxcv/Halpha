# Halpha 个人交易工作台交互设计

**文档编号：** HALPHA-UX-002  
**层级：** L3  
**L3 类型：** DOMAIN  
**主要语义所有者：** UX  
**所属实现模块：** Web 前端与应用 API  
**语言版本：** zh-CN  
**上位文档或条款：** HALPHA-UX-001  
**直接依赖：** HALPHA-TRADEPLAN-002、HALPHA-DAT-002、HALPHA-CAP-002、HALPHA-EXE-002、HALPHA-OUT-002  
**直接消费者：** Web 产品实现  
**适用纵向约束：** HALPHA-ENG-001、HALPHA-SYS-001  
**本文档负责：** 工作台的信息组织、状态表达、命令反馈、安全交互和视觉底线  
**本文档不负责：** 领域状态机、资金计算、交易所变更请求、当前页面数量或精确组件版本  

## 1. 设计结论

工作台服务于单一所有者完成计划、激活、观察、控制、核对和复盘。界面应直接显示会改变决定的信息，不建立通用任务、审批、回执或通知平台。

当前路由、页面数量和主题参数属于 L4 或实现，不在本文件固化。长期信息结构按用户任务划分为计划、当前激活与操作、账户与环境三个区域；页面和任务较少时可以合并呈现。

## 2. 共同交互原则

- 服务端事实是权威来源，浏览器缓存不得推进领域状态；
- 状态改变必须显示明确结果和最新服务端版本，不做乐观成功；
- 请求超时或结果未知时，按原请求身份查询，不重新提交；
- Demo 与 Live 的行情、账户、资金、动作和结果始终清晰可辨；
- 金额、数量、价格和比率以精确十进制文本输入和显示；
- 服务端与持久事实保留 UTC 时间，所有用户可见时间统一转换为 `Asia/Shanghai`（UTC+8）并带时区标识；
- 未知、过期、冲突、部分成交和保护缺口必须直接可见；
- 首屏面积与信息对当前决定的价值相称；低价值标题、背景说明、空状态解释、重复身份和技术元数据不得以大块容器、大段留白或重复声明占据主要区域，宜压缩为短标签、单行状态或按需展开；
- 可增长列表默认只显示最近的有限条目，并通过显示更多或分页继续读取；
- 高影响操作要求用户看到对象、范围和后果，但不叠加无业务意义的多层确认。

命令结果统一表达为 `APPLIED | REJECTED | UNKNOWN`，并提供原因、当前版本和可执行的下一步。

信息载体采用以下最小选择：

- 需要同时比较、共同决定或连续操作的当前内容放在同一页面；
- 同一任务内互斥的视角、状态范围或当前与历史内容使用页签，不在一个滚动流中上下堆叠；
- 当前对象的解释、参数和次要证据使用局部展开、抽屉或浮层，在不离开任务的情况下查看；
- 弹窗只用于必须打断当前操作的后果确认或短时决定，不承载普通浏览和长篇说明；
- hint 或 tooltip 只解释短术语、单位和控件，不承载风险、未知、失败或唯一事实；
- 需要稳定地址、独立恢复、跨入口访问或大量深层证据时才使用独立页面。

工作台采用一套固定的亮色视觉体系，不提供多主题切换：

- 浅灰操作轨承载品牌、环境和主导航，近白顶栏承载当前页面上下文，近白内容面承载交易事实与操作；三者保持稳定位置和清楚边界；
- 文字按决策价值使用主文字、次要文字和弱化文字三级中性色：结论、关键状态和主要数值使用主文字，说明与上下文使用次要文字，标签、时间和技术元数据使用弱化文字；不得把全部文字处理成同一黑色、字号和字重；
- 暖黄色是结构强调色，只用于当前导航、当前页签、区域内主要行动和键盘焦点；需要文字形式的次级操作时使用与暖黄色同属当前视觉体系且满足对比度的深色强调文字，不铺满普通面板；
- 成功、警告、危险和信息分别使用固定的绿色、黄色、红色和蓝色语义色族，不随结构强调色变化。每个语义色族统一拥有图标或边界色、浅背景和可读文字色；普通提示优先使用浅背景或中性表面配语义边界，只有必须立即识别的危险状态才使用大面积实色，并始终配合文字、图标或原因；
- 页面标题、区域标题、关键状态或数值、正文、标签与时间形成递减且稳定的字号和字重层级；强调色和大字号只提升当前判断所需内容，不用于普通说明或技术元数据；
- 买卖方向和盈亏颜色不得替代业务标签；
- 交易方向和盈亏使用红绿配色辅助快速辨认，默认红跌绿涨；用户可在设置中一键切换为红涨绿跌，选择在当前浏览器保存并作用于全部工作台页面。做多、做空和盈亏始终同时保留文字或正负号，零值与未知值不着涨跌色；
- 视觉层级优先由表面色差、间距、对齐、字号和字重建立；同一选择或层级只使用一个主要强调信号，填充色已经表达选择时不再叠加常驻边框；
- 边框只用于输入、数据比较、弹窗和确需界定范围的结构或操作，不使用装饰性整行分割线，不让嵌套、相邻或并列边框重合并重复表达同一边界；键盘焦点轮廓作为临时交互反馈独立保留；
- 常规面板不使用重阴影，不使用渐变、发光、玻璃效果、装饰色块或卡片套卡片；
- 页面标题、页签、按钮、输入、列表行和事实网格保持紧凑尺度与统一几何；金额、价格、比率、时间和标识使用等宽数字，便于纵向比较；
- 每个可操作控件具有默认、悬停、按下、焦点、选中、处理中、禁用和结果反馈中实际适用的状态；按下反馈轻微且即时，动效只帮助识别状态变化并尊重减少动效偏好；
- 桌面保持稳定操作轨和工作区，窄屏将主导航移入临时抽屉并按原任务顺序单列化；优先减少列数和次要说明，不缩小关键事实或隐藏停止、退出和接管。

## 3. 计划与激活

用户应能够为计划填写区别于策略或订单模式名称的自定义名称，选择 `STRATEGY_SIGNAL | DIRECT_EXECUTION` 决策依据，编辑计划草稿、查看校验问题、固定版本、预览激活范围并明确激活。计划列表以计划名称为主要身份，并紧凑标注决策依据、订单计划摘要、创建时间以及人工或 AI 创建来源；来源和创建时间未知时明确显示未知。固定后内容只读；修改时创建新版本。

未固定草稿提供删除入口，使用危险语义按钮并在弹窗中显示计划名称、不可恢复后果和不会影响策略定义或其他计划的范围。弹窗的执行按钮明确写“删除草稿”；提交期间防止重复，只有服务端确认删除后才从列表移除，版本冲突、已固定、失败或结果未知时保留草稿并显示原因。固定计划不显示删除入口。

选择 `STRATEGY_SIGNAL` 时，按需显示策略清单提供的价值逻辑、适用场景和实际执行行为，使策略增多后仍可理解当前选择。选择 `DIRECT_EXECUTION` 时明确说明激活可运行后无需等待策略信号，但仍会经过当前事实、CAP 和 EXE；不得显示虚构策略。已创建计划显示本次交易金额、期限、条件、订单计划、动态管理和保护配置，以及策略依据适用时由策略清单声明的关键参数和值；页面不以策略介绍替代计划配置，不猜测未声明的关键参数，也不另写一套参数标签、单位或格式。

订单计划编辑与当前计划保持同一任务页面，不建立第二交易终端。页面按环境、工具与方向，条件，价格和金额分布，场所订单政策，动态管理，保护与退出的顺序组织；默认关闭的动态规则按需展开，但启用后的关键阈值和后果始终显示在计划摘要中。

浏览器可以即时提示格式错误，但固定和激活使用服务端归一化预览。用户可在同一编辑区选择单笔或阶梯；单笔市价和 `priceMatch` 明确显示用于数量计算的当前参考价以及成交价仍未知。预览至少按档位显示原始与归一化价格、名义金额、数量、方向、订单类型、有效方式、maker-only、档位问题和累计金额，并显示工具规则来源与事实截止点。区间倒置、权重数量不匹配、非正权重、精度归一化后档位重合、最小数量或名义金额不满足、累计金额越界以及场所参数冲突必须定位到对应字段或档位；不得静默删除、合并或改写用户档位。

K 线图可以作为同一订单计划编辑区的输入加速器，但不是第二份订单模型或事实源。主图只接收与页面当前 `environment_id` 和预期来源匹配的 DAT 输出，并持续标明环境、来源和新鲜度；历史 K 线、实时更新、盘口、数量计算、订单预览和场所动作必须属于同一环境，不得把 Demo 与 Live 数据并列、拼接或互相补齐。当前环境的 K 线缺失、过期、断连或来源不匹配时，主图直接显示不可用或未知，清除旧图线和草稿价格投影，并阻断依赖该价格的预览与增险操作，不回退到另一环境。切换环境时关闭旧订阅并等待新环境历史与实时来源确认，不能在切换期间沿用旧价格。

图上纵向拖动区间、移动草稿边界或修改数字字段都写回同一组计划草稿字段；拖动结束后才请求服务端归一化，只有当前版本预览通过时才显示带“草稿”身份的标准化档位线，输入陈旧、预览失败或被阻断时隐藏旧档位线或明确标为陈旧。支撑、阻力和趋势线属于页面会话内的分析绘图，必须与草稿投影、固定计划及场所已确认订单使用文字身份和线型共同区分，不能只靠颜色。每条图线提供等价数值列表，数字输入保留为完整键盘替代路径；`Esc` 可以取消未完成绘图或撤销最近一次图上草稿修改。窄屏优先保证查看、数字输入和档位复核，不要求承载桌面精度的多档拖动。

分析绘图不得直接移动真实入场、止盈或止损。若未来允许沿趋势线动态管理，图表只提交固定的时间—价格锚点和有界规则草稿；TradePlan 与 EXE 仍负责计算时钟、价格步进、最大调整次数、止损只收紧、撤单确认后替换、部分成交竞态、未知结果和重启恢复。在这些类型和运行时消费者获得当前 L4 资格前，界面只能将趋势线显示为分析对象，不得暗示已经形成自动交易规则。

当前可操作计划与历史计划使用同一计划任务中的独立页签；历史内容不占用当前计划的主扫描区域。

激活预览至少显示环境、账户、工具、方向、决策依据、计划交易金额、归一化订单档位、允许动作、动态管理、有效期、保护与退出规则、当前预览摘要及已知限制。工具规则或预览依据已经变化时要求重新预览，不沿用旧摘要。激活按钮只让该固定计划开始运行，不表示已提交或成交；`DIRECT_EXECUTION` 会在激活进入可运行状态后形成计划内动作，必须在按钮后果中明确说明。

工作台不提供独立资金授权、账户额度或资金分配页面。真实账户交易计划的激活可以要求一次明确确认，但不再要求额外资金授权对象或逐动作审批。

## 4. 当前激活与操作

当前激活区域应优先显示：

- 生命周期、是否允许新增风险和最近事实时间；
- 持仓、开放订单、待定动作及未知结果；
- 订单计划的总档数、已建立责任、已提交、开放、部分成交、已撤销、已拒绝和未知档位；
- 保护是否存在、是否完整以及缺口；
- 当前损失与激活边界的关系；
- 最近关键事件、核对结果和复盘入口。

总览以当前敞口和订单责任为默认内容。尚未成交时优先显示按 `schedule_ref` 聚合的计划档位，并能展开每档的计划条款、执行动作和场所事实；出现持仓后至少显示交易对、方向、数量、入场价、可确认的当前价格与盈亏、保护、管理该仓位的计划及适用策略依据和事实刷新时间。当前权威事实没有提供的强平价、保证金率等字段保持未知，不由页面推算。最近交易结果放在独立页签，不与当前责任混排。

交易记录列表至少显示交易对、方向、时间、交易金额和净盈亏。交易详情在权威事实存在时继续显示决策依据及适用时的策略、操作理由与触发来源、逐笔价格和数量、手续费、挂单或吃单以及持仓周期；缺失项明确显示未知，不从订单状态、计划或近似时间猜测。

停止新增风险、退出计划和用户接管必须并列可见，并清楚说明不同后果。停止新增风险不能误阻撤销、保护和减仓；退出计划会撤销仍在开放的增险档位并处理已有风险，但命令被接受不代表撤单或平仓已经生效。用户接管后界面转为只读核对，并提供场所官方接管指引。

## 5. 账户、环境与故障

账户与环境区域显示当前环境、连接状态、事实新鲜度、凭据是否可用、进程状态和恢复结果，但不得显示秘密值。

当前环境和账户由稳定的全局载体持续标识时，任务区域不重复铺陈相同身份；只有它会改变当前操作、解释失败或提示上下文冲突时才局部重申。行情、账户、资金、动作、订单、成交和其他产品事实一次只呈现当前环境的数据；页面外壳与局部数据的环境或账户不一致时拒绝展示和增险操作。只读行情也不构成跨环境例外。

后端、数据库、场所或前端局部不可用时，工作台应保留能够安全呈现的只读操作信息。无法证明当前性时，禁用增险操作并显示原因，不以空白页面或笼统错误代替。

## 6. 安全与可访问性

状态改变要求已认证会话、来源校验和防跨站请求保护。密码与秘密不进入 URL、浏览器持久存储、页面日志或错误详情。

界面以桌面为主并支持窄屏阅读。关键控件必须有文字标签、键盘焦点、错误关联、足够对比度和状态播报；颜色不能单独承担含义。避免闪烁行情、装饰动画和游戏化反馈。

## 7. 浏览器验证

根据改动影响验证真实浏览器行为，至少覆盖相关的：

- 计划校验、固定、版本只读和激活预览；
- 策略信号与直接执行的区分、服务端订单归一化、档位聚合和场所政策冲突；
- 固定、线性、指数和显式权重组合，以及动态改单、撤单和移动保护的触发与边界；
- 重复点击、超时、刷新、陈旧页面和未知结果；
- 环境隔离、切换后旧行情清除、保护缺口、部分成交和用户接管；
- 停止、退出与接管的不同结果；
- 未认证、会话到期、来源校验和跨站请求反例；
- 桌面、窄屏、键盘、焦点、对比度与状态播报。

浏览器检查证明交互行为，不代替数据库、场所或策略的直接验证。
