# Halpha 单所有者交易工作台交互与视觉标准

**文档编号：** HALPHA-UX-002  
**层级：** L3  
**L3 类型：** DOMAIN  
**主要语义所有者：** UX  
**所属实现模块：** `user_workbench`  
**语言版本：** zh-CN  
**上位文档或条款：** HALPHA-UX-001  
**直接依赖：** HALPHA-ALP-002、HALPHA-TRADEPLAN-002、HALPHA-DAT-002、HALPHA-CAP-002、HALPHA-EXE-002、HALPHA-OUT-002  
**适用纵向约束：** HALPHA-SYS-001、HALPHA-ENG-001  
**本文档负责：** 单所有者 Web 工作台的三个主要入口；计划配置与激活；运行状态与三项控制；账户连接与事实；环境、过期、未知和幂等结果呈现；最小组件、安全、视觉、响应式与浏览器验证要求  
**本文档不负责：** 定义策略、计划、资金、事实、订单、保护、核对或复盘业务语义；持有交易所凭据；规定当前组件精确版本、部署状态或检查结果；建设通用任务、回执、通知、恢复或专业交易终端平台  

---

# 0. 交互结论【UX-AUTO-SUM-001】

工作台只围绕三个主要入口组织：计划配置与激活、一个激活的运行状态、账户与环境连接。用户配置并固定计划后，在同一页面复核环境、账户、工具、资金限制、动作范围和期限，再以一次“激活计划”命令创建包含机器执行授权的 `PlanActivation`。没有账户上限页、第二授权、外置配置门或逐动作确认。

运行页持续显示 Halpha 已知事实、未知、开放责任和保护情况，并提供停止新增风险、退出策略、用户接管三项控制。停止新增风险后，同一激活不能恢复；继续交易需要新建激活。工作台不建立通用任务、命令回执或通知记录族，不提供第二操作页面或恢复状态机。

界面必须区分计划、请求是否已经应用、`ExecutionAction`、场所事实和最终责任闭合。HTTP 成功、按钮禁用、动作已提交、场所已接受、部分成交、持仓为零和责任已结束都不是同一结果。

---

# 1. 组件实现方案【UX-AUTO-CMP-001】

## 1.1 采用组件及使用边界【UX-AUTO-CMP-001-REQ】

前端使用 React 与 MUI 组合页面，RJSF/AJV 根据策略 schema 生成并校验参数表单，TanStack Query 管理浏览器内查询，API 类型和客户端从服务器 OpenAPI 生成。组件状态只用于呈现；服务器领域对象和校验结果始终是权威。

Halpha 只自研交易字段标签、少量 Decimal/单位控件、页面组合和服务器错误映射。不建设组件库、表单引擎、缓存层、API SDK、专业图表、通知中心或通用工作流。当前精确版本和配置由 L4 记录。

## 1.2 浏览器状态限制【UX-AUTO-CMP-002-REQ】

每个事实视图显示服务器事实截止点和页面取得时间。轮询中断、页面离线、服务器错误或截止点停止推进时，显著显示“视图可能过期”，并禁止激活；停止新增风险、退出和用户接管仍可提交，但页面只能显示服务器返回的确定结果或 `UNKNOWN`。

浏览器不离线排队、不乐观更新业务状态，也不在 localStorage 或 Service Worker 保存激活资金限制、授权或控制结果。请求超时后使用原幂等请求身份查询，不创建第二请求猜测第一次是否成功。

---

# 2. 幂等请求与结果【UX-AUTO-REC-001】

工作台不拥有业务记录族。每个会改变状态的请求携带稳定 `request_id`、目标身份、预期版本和不可变内容；拥有领域将该身份写入自己的 `PlanEvent` 或 `PlanActivation`。重复身份和相同内容返回原结果，相同身份但内容不同返回冲突。

页面只使用三个结果：`APPLIED | REJECTED | UNKNOWN`，并重新读取业务对象：

- `APPLIED` 表示相应领域变化已经持久化，不表示场所动作已经成交或责任已经闭合；
- `REJECTED` 表示确定没有应用该请求，并显示理由；
- `UNKNOWN` 表示处理结果无法确认，页面按原身份查询且不允许重复提交。

涉及场所动作时，只有 `ExecutionAction` 和 `VenueFact` 能说明提交、成交、保护和核对结果。浏览器、邮件或日志都不能形成第二业务状态。

# 3. 三个主要入口【UX-AUTO-IA-001】

## 3.1 全局框架【UX-AUTO-IA-001-REQ】

所有页面顶部固定显示：`Demo/Live`、账户、工具、事实截止点、新增风险是否已停止、当前未知和开放责任。环境同时使用文字、图标和色彩；切换环境或账户后必须重新读取数据并清空待提交表单。

主导航只有：

| 路由 | 用户任务 |
|---|---|
| `/plans` | 选择策略、配置参数、固定计划并复核激活 |
| `/activations/:activation_id` | 查看运行状态、订单/持仓/保护和使用三项控制 |
| `/settings/accounts` | 配置和查看环境、账户连接及场所账户事实 |

计划历史和运行时间线在对应计划或激活内展开；不另建总览、资金中心、通知中心、复盘中心或第二操作入口。需要专业行情分析时使用外部工具，Halpha 只保留必要引用和返回上下文。

---

# 4. 计划配置与激活【UX-AUTO-PLAN-001】

## 4.1 策略与参数【UX-AUTO-PLAN-001-REQ】

计划页列出产品代码中登记的策略身份、版本、逻辑摘要、参数、所需事实、允许动作、适用范围和已知限制。RJSF 按 ALP schema 生成表单；Decimal 使用文本和明确单位，不经 JavaScript `number` 隐式转换。客户端提示不替代服务器校验。

页面按四组显示：策略参数；环境、账户、工具和方向；入场、保护与退出规则；最大保证金、最大名义仓位、最大允许损失、单动作上限、动作范围和期限。提交固定前只是页面输入；固定后形成只读 `TradePlanVersion`，修改必须形成新版本。

## 4.2 激活复核【UX-AUTO-ACT-001-REQ】

激活前同屏显示计划和代码版本、环境/账户/工具、当前事实截止点、入场与退出、最大保证金、最大名义仓位、最大允许损失、单动作上限、动作范围、期限、是否存在同账户未闭合激活及所有未知。

页面明确说明：资金限制只约束 Halpha 主动增加风险，不是交易所资金隔离或损失保证；最大损失只计算本激活可归属的净结果，触发后停止新增风险并退出；激活后 Halpha 可在列明范围内持续执行；激活成功不表示已经下单、成交或获利。

主按钮为“激活计划”。已登录用户查看一次完整后果预览后显式提交即可；不增加第二确认或重新认证步骤。一次请求只创建一个 `PlanActivation`，不直接调用交易所。双击、超时和刷新复用同一请求身份；结果未知时只提供原身份查询。

---

# 5. 运行状态与时间线【UX-AUTO-RUN-001】

## 5.1 运行页【UX-AUTO-RUN-001-REQ】

运行页按以下顺序展示：激活状态与事实截止点；环境、账户、工具和方向；资金限制与当前使用；条件判断；持仓、开放订单与保护；动作未决与未知；三项控制；时间线。存在持仓且保护未确认时，在页面顶部显示不可忽略的提示。

条件只显示 `TRUE | FALSE | UNKNOWN` 和理由。`UNKNOWN` 不能显示成“未触发”。部分成交同时显示已成交量、剩余量、当前持仓和保护状态；订单状态不能替代持仓事实。

## 5.2 唯一时间线投影【UX-AUTO-RUN-002-REQ】

时间线只读取 `PlanActivation`、`PlanEvent`、`ExecutionAction` 和 `VenueFact`，按“计划决定 → 执行动作 → 场所事实 → 核对结果”展示。页面不创建额外时间线对象；来源顺序或环境无法确认时显示 `UNKNOWN`，不由前端猜测完成。

Demo 与 Live 使用同一状态词，但每一行都显示环境和账户。Demo 结果只说明模拟环境中的流程和策略行为，不证明 Live 的流动性、滑点、费用、权限、可用性或未来收益。

---

# 6. 三项控制【UX-AUTO-CTL-001】

| 控制 | 页面说明 | 显示为已应用的条件 |
|---|---|---|
| 停止新增风险 | 禁止新的开仓、加仓和入场重挂；继续保护、撤单、减仓和退出；同一激活不可恢复 | 服务器重读显示停止事实已持久化；不宣称持仓已平 |
| 退出策略 | 停止新增风险并按当前事实撤单、减仓或平仓 | 激活已经进入退出；场所结果按每个动作继续显示 |
| 用户接管 | Halpha 停止该激活新的场所写，用户到交易所官方入口处理；不自动撤单或平仓 | 接管范围已持久化且策略实例已停止；在途结果仍显著显示 |

三项控制使用不同按钮、后果预览和幂等请求身份。保护缺口、动作未决或 Executor 不可达时，不得显示“已经安全停止”或“已经退出”。用户接管后只显示只读核对和新建计划入口，不显示恢复机器写入。

---

# 7. 账户连接与结束【UX-AUTO-CAP-001】

账户设置页只显示环境、账户连接状态、场所账户事实、数据截止点和外部接管入口，不建立 Halpha 账户资金上限或分配页面。每次激活的资金限制只在计划和激活页面查看或设置；外部余额变化不能自动修改这些限制。

场所事实使已有持仓自然越过激活限制时，页面显示“阻止后续增险，不自动退出”；保护、减仓和退出仍按计划与场所事实处理。

激活只有在服务器显示场所持仓为零、相关订单和执行动作都已确定结束、没有写入结果未知或归属冲突时才显示“已结束”。策略停止、用户接管、局部订单终态或页面余额为零都不能单独显示结束。

---

# 8. Web 安全【UX-AUTO-SEC-001】

工作台只认证单一所有者，使用成熟口令哈希、短时签名会话、CSRF、可信 Host、同源检查和基本登录限频。`GET/HEAD`、资源预取和跨站请求不得改变状态。精确参数由 L4 记录。

认证成功只允许进入工作台，不自动授权激活或场所动作。激活、退出和用户接管都需要当前后果预览与明确提交。浏览器响应、DOM、日志和剪贴板不得出现交易所、数据库或会话秘密；安全组件失效时拒绝受影响命令，并提示使用交易所官方入口接管。

---

# 9. 视觉、响应式与可访问性【UX-AUTO-VIS-001】

界面使用中性基底、单一交互强调色和少量语义色。Live、未知、保护缺口和用户接管同时使用文字、图标与色彩；盈亏颜色不表示建议或安全。数字、单位、正负号和时间对齐稳定，不使用行情闪烁、游戏化或装饰性动效。

主要支持桌面；窄屏可折叠导航和转换表格，但激活后果、资金限制、保护缺口和三项控制不能隐藏。关键控件支持键盘、明确焦点、标签、错误关联、足够对比度和状态播报。优先使用 MUI 现成组件，没有当前任务消费者的领域控件不建设。

---

# 10. 浏览器验收【UX-AUTO-TST-001】

浏览器测试按当前影响至少覆盖：

1. 参数、Decimal、服务器错误和固定版本不可编辑；
2. 一次激活完整显示范围和后果，双击、超时与刷新不产生第二激活；
3. Demo/Live、过期、`UNKNOWN`、动作未决、部分成交、持仓和保护不会混淆；
4. 停止新增风险、退出和用户接管后果不同，停止不误阻保护与减仓，且不存在恢复命令；
5. 陈旧或离线页面不能激活，未知请求只查询原身份；
6. 未认证、会话到期、CSRF、Host/Origin 和跨站反例不能改变状态；
7. 桌面、窄屏、键盘、焦点、对比度和状态播报满足实际使用需要；
8. 三个主要入口之外不存在通用任务、回执、通知、第二操作或恢复平台。

浏览器测试只证明交互契约，不证明交易所、数据库或策略当前可用；当前结果由 L4 记录。

---

# 11. 复杂度与 L4 边界【UX-AUTO-CPLX-001】

前端保持一个 Web 应用和一个 API，不新增通用任务、回执或通知平台、第二操作界面、进程、数据库、写入路径或业务事实权威。

L4 只记录当前精确组件版本、构建摘要、安全参数、主题变量、目标浏览器与屏幕、三个入口的建设状态、浏览器测试结果和已知限制。本 L3 不声明这些当前已经部署或可用于真实资金。
