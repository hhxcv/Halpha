# Halpha 单次激活复盘契约

**文档编号：** HALPHA-OUT-002  
**层级：** L3  
**L3 类型：** DOMAIN  
**主要语义所有者：** OUT  
**所属实现模块：** `planning` 中的复盘分区；不建立独立服务、进程或分析平台  
**语言版本：** zh-CN  
**上位文档或条款：** HALPHA-OUT-001  
**直接依赖：** HALPHA-TRADEPLAN-002、HALPHA-DAT-002、HALPHA-CAP-002、HALPHA-EXE-002  
**直接消费者：** HALPHA-UX-002  
**适用纵向约束：** HALPHA-SYS-001、HALPHA-ENG-001  
**本文档负责：** 一个 `Review` 记录族；单次激活复盘的身份、引用、版本、写入、修正、失败和验证  
**本文档不负责：** 修改来源记录或相邻责任；自动因果归因、策略评分、待办、改进交接、审批、worker 或分析平台  

---

# 0. 设计结论【OUT-RT-SUM-001】

OUT 只保存 `Review`。每个 `PlanActivation` 有一个稳定复盘身份和不可变版本链；成交、非成交、未知和接管使用同一结构。后续问题只是 `Review` 内的可选文本，不形成第二记录族或自动流程。

# 1. 身份与版本【OUT-RT-SCP-001】

`Review.review_id` 由 `activation_id` 稳定派生。同一激活的重复写入返回同一当前版本；相同预期版本但内容不同返回冲突，不得生成并行复盘。

后来事实或用户评价发生实质变化时，追加版本并引用前一版本、变更原因和新的事实截止点。旧版本不可修改或删除。相邻事实的修正仍由原领域完成，OUT 只更新自身评价。

# 2. 最小内容【OUT-RT-REV-001】

`Review` 只保存或引用会改变评价的内容：

- 复盘、激活、计划、环境、账户、工具、版本和记录时间；
- 评价使用的 `VenueFact`、`ExecutionAction` 与其他来源引用及事实截止点；
- 成交、非成交、未知或接管结果，以及费用和可可靠归属的盈亏；
- 激活资金限制、机器执行范围、停止、保护、退出与核对是否按设计工作；
- 关键偏差、未知、来源缺口和仍由相邻领域承担的开放责任；
- 用户的简短结论，以及可选的后续问题、依据、影响范围和建议语义所有者。

未知值保持未知，不填零、不补造成交或费用。无法精确归属的接管后结果只记录账户净事实和限制，不能写成 Halpha 激活结果。

# 3. 形成与修正【OUT-RT-WRT-001】

当一个激活已结束、明确不会行动、已接管，或虽有未知但评价边界已经清楚时，拥有模块可在处理该结果的同一协调调用后记录复盘。复盘不需要等待所有开放责任关闭，也不得反向推进激活或动作状态。

公开能力只需：按激活记录或修订复盘；读取一条复盘及版本；按时间列出已有复盘。写入使用 `activation_id`、预期版本和稳定请求身份保证幂等，不建立异步队列、领取、游标或扫描 worker。

用户需要实际处理后续问题时，在复盘之外明确开始普通研究、设计或开发工作。OUT 不创建任务，也不维护处理状态。

# 4. 失败与隔离【OUT-RT-FAL-001】

| 情况 | 结果 |
|---|---|
| 来源缺失、冲突或过期 | 记录受影响未知和缺口，不猜测 |
| 同一激活已有不同内容的当前版本 | 返回版本冲突，不覆盖 |
| OUT 写入失败 | 不改变计划、资金限制、权限、动作、保护、核对或接管 |
| 后来事实修正 | 追加复盘版本，不改写旧版或源事实 |
| Demo/Live 身份冲突 | 拒绝复盘，不跨环境拼接 |

复盘失败不得阻止保护、退出、核对和用户接管。复盘成功不得关闭相邻领域责任、恢复旧激活新增风险或扩大资金与机器执行范围。

# 5. 验证与复杂度【OUT-RT-CHK-001】

至少验证：

1. 一个激活只有一个复盘身份，重复请求不生成第二条；
2. 成交、未成交、未知和接管都能形成如实复盘；
3. 未知费用、盈亏和归属不被填零或伪造；
4. 后来修正追加版本并保留旧版；
5. Demo 与 Live 不能交叉引用；
6. 复盘写入失败不改变交易责任；
7. 后续问题只保存在 `Review`，不会自动创建任务或修改目标领域；
8. 运行中只有一个 OUT 记录族，没有独立 worker、队列、游标或分析存储。

L4 只记录当前复盘启用范围、实际字段和直接验证。OUT 不新增进程、数据库、场所写路径或运行模式。
