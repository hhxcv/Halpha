# Halpha 激活资金检查契约

**文档编号：** HALPHA-CAP-002  
**层级：** L3  
**L3 类型：** DOMAIN  
**主要语义所有者：** CAP  
**所属实现模块：** capital 的无状态检查能力  
**语言版本：** zh-CN  
**上位文档或条款：** HALPHA-CAP-001  
**直接依赖：** HALPHA-TRADEPLAN-002、HALPHA-DAT-002  
**直接消费者：** HALPHA-EXE-002  
**适用纵向约束：** HALPHA-SYS-001、HALPHA-ENG-001  
**本文档负责：** 读取激活资金边界并对动作分类、计算和检查  
**本文档不负责：** 定义激活字段或生命周期、授予用户决定、执行场所动作和选择当前金额

## 1. 设计结论

CAP 不拥有持久业务记录。对于半自动计划，它读取 TRADEPLAN 已固定的交易金额、动作范围和 DAT 当前事实，返回确定的检查结果；消费方把结果持久化到自己拥有的记录。账户额度、资金分配或机器授权不是这条路径的输入。

资金仍留在交易所账户中。检查结果不构成交易所冻结、账户分配或损失保证。

## 2. 输入与输出

输入包括激活身份与版本、环境和账户、适用工具、计划交易金额、允许动作和期限，以及当前持仓、开放订单、在途动作、余额、价格、数量、费用和场所规则等必要事实。

结果为：允许增加风险、允许不增险动作、拒绝，或未知。每个结果带稳定原因和事实截止点。CAP 不返回新的授权对象或状态机。

## 3. 动作分类和检查

增加或可能转换风险的动作，必须满足激活有效、未停止新增风险、动作属于范围、事实当前、单动作和累计敞口不超过该计划的交易金额、激活损失未触发计划保护条件以及场所当前可承载。

只有能够基于当前持仓、开放订单和最坏可执行数量证明不会反向开仓的撤销、保护、减仓和平仓，才按不增险处理。无法证明时返回未知，不允许发起交易所变更请求。

TRADEPLAN 形成动作意图时调用一次；EXE 向交易所提交变更请求前刷新事实并调用同一函数。后一次可以否决前一次，但不创建第二审批或记录。

## 4. 损失口径

激活净结果使用可归属于该激活的已实现盈亏、未实现盈亏、实际资金费和实际手续费：

```text
激活净结果 = 已实现盈亏 + 未实现盈亏 + 资金费 - 手续费
激活损失 = max(-激活净结果, 0)
```

达到用户设定的允许损失后，CAP 返回触发结果，由 TRADEPLAN 在激活生命周期中记录停止并进入退出。后来盈利不恢复已经触发的新增风险能力；归属未知时停止新增风险并核对。

## 5. 故障与验证

数据库不可用、激活版本冲突、事实过期或归属不明时不以内存值继续，返回未知。系统恢复顺序由 SYS 拥有；CAP 只对恢复后的当前输入重算。

至少验证累计敞口、部分成交、重复输入、损失口径、停止后保护和退出仍可用、Demo/Live 隔离，以及形成动作责任与发起交易所变更请求前使用同一检查。当前字段映射、金额和阈值由代码与 L4 记录。
