**本文档负责：** 代码策略身份、参数、输入、输出及量化组件使用边界  
**本文档不负责：** 交易计划、资金检查、交易所变更请求、当前策略参数和组件精确版本

## 1. 设计结论

Halpha 使用成熟量化组件承担行情回调、时钟、指标、策略生命周期和回测。Halpha 只保留随产品代码发布的策略清单、参数校验、纯策略逻辑和薄适配，不建设第二策略框架、指标库、撮合器或动态插件平台。

需要项目自行研究和保有、外部没有可直接复用公开实现的 Alpha 因子与决策逻辑，可以作为纯策略逻辑自研；其行情、时钟、通用指标、回放、策略生命周期和执行基础仍复用成熟量化组件，不因 Alpha 定制扩大为自研基础框架。

策略接收规范输入并产生零个或一个瞬态策略提议。策略提议不是记录、计划、授权或场所命令；TRADEPLAN 是唯一消费者。

## 2. 策略定义

每个策略固定身份、代码版本引用、参数 schema、所需输入、适用范围、可提出的动作种类和事实未知时的不输出规则。策略清单同时提供面向用户的价值逻辑、适用场景和实际执行行为说明，以及已创建计划用于展示的关键策略参数定义；关键参数定义只声明参数键、用户标签、单位和最小显示格式，不复制计划值或页面布局。说明和显示定义只解释同一策略定义，不建立第二份策略规则。逻辑、参数含义、输入或动作语义改变时形成新策略版本；运行中的参数不热更新。

研究候选只有在项目所有者明确选中后才成为产品实现输入。产品消费其框架无关的固定决策说明和小型输入输出样例，重新实现或审查纯策略逻辑；不得导入 VectorBT、Notebook、研究缓存或研究目录来运行策略。

策略只从产品代码的显式清单装载，不扫描目录、不下载或执行任意代码。策略逻辑不得取得订单工厂、执行客户端或等价交易所变更请求入口。

## 3. 策略提议

策略提议保存足以稳定识别激活、策略、触发、输入摘要、拟议内容、理由和有效期的信息。相同触发身份和输入产生相同结果；同一身份出现不同输入摘要时返回冲突，不生成第二提议。

输入缺失、过期、不连续、冲突、组件异常或数值无效时，不提出新增风险。停止新增风险后只允许形成计划仍需要的保护或退出方向提议；用户接管后停止产生提议。

## 4. 在线与离线

实时和离线使用同一纯策略逻辑、参数规范化和提议契约。历史数据缺少实时事实时可以使用明确标注的代理输入，但必须说明差异，不能维护第二策略实现来制造等价。

候选进入产品时，先在相同规范输入和截止点上核对研究交接样例与产品决策；随后由产品量化组件验证事件、订单、成交、资金费、保证金和场所语义。研究与产品收益不要求机械相等，但策略决策差异和执行模型差异必须分开定位，不能解释时不启用该策略。

组件承担其实际支持的回放、撮合、费用、滑点和报告；未建模项保持明确。回测和模拟结果不证明未来收益或真实资金可用性。

## 5. 恢复与验证

策略运行状态可以使用组件内存，但被采纳的触发和决定由 TRADEPLAN 保存，外部结果由 DAT/EXE 保存。系统恢复时，策略适配器只按 SYS 已确认可继续的激活和事实重建；不能等价重建时不输出新增风险。

至少验证参数边界、warmup 与数据截止点、重复与冲突输入、未知不增险、研究交接样例与产品决策一致、在线离线提议一致，以及策略代码无法发起交易所变更请求或依赖研究运行时。当前策略、组件版本和实际结果由 L4 记录。
