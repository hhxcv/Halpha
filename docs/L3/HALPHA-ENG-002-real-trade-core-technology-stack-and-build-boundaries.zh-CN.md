## 1. 技术结论

Halpha 长期采用 Python 后端、React Web 前端、PostgreSQL 持久存储和 NautilusTrader 量化交易栈。成熟组件承担 HTTP、schema、事务、迁移、前端通用组件、行情、指标、回测、订单模型和场所适配；Halpha 只实现自身产品语义和必要边界。

当前精确版本、锁文件、平台结果和配置由 L4 指向代码与配置来源，不在本契约复制。

## 2. 量化组件边界

NautilusTrader 承担行情回调、时钟、指标、策略运行、订单对象和场所客户端。Halpha 不实现第二行情泵、指标引擎、回测撮合、订单状态机或交易连接作为后备。

量化交易基础核心对稳定性、性能和失败语义要求高，默认复用 NautilusTrader 的公开能力。在项目复杂度可接受范围内，先理解并验证与当前结果有关的组件设计、生命周期和扩展点，再通过受支持扩展或薄适配接入 Halpha；不得以局部定制为由自研替代组件已经承担的基础核心框架。

组件缓存、组合和风险投影不是 Halpha 的计划、资金、事实或动作权威。涉及资金边界和归属的数值进入 Halpha 领域前使用确定精度转换并验证单位。

组件升级需检查公开行情与执行契约、订单身份和查询、实时与离线策略输入、数值和时间边界以及目标主机运行。公开能力不足时停止或明确不支持，不自动切换第二实现。

## 3. 应用和数据组件

Web 后端和前端组件只承担通用传输、校验、事务、迁移、呈现和查询缓存。产品路由调用拥有领域；浏览器缓存、日志、邮件和组件状态都不是业务权威。

数据库、迁移或 schema 不可用时，不得形成依赖它们的业务状态，也不得发起相应交易所变更请求。前端不可用时，用户仍能取得交易场所官方接管入口。

## 4. 运行 profile

- DEMO：运行产品闭环并连接模拟场所，禁止真实账户交易凭据和真实账户交易动作。
- LIVE_READ_ONLY：只使用公开行情和可删除输出，不取得产品写、私有账户或执行能力。
- LIVE_WRITE（真实账户交易）：使用同一产品闭环执行用户已启用计划，仅在当前前置事实满足时取得真实账户交易凭据并发起交易所变更请求。

环境差异由受控配置选择，不能散落为第二业务实现。存储和运行实例的当前隔离方式由 SYS 详细设计与 L4 共同记录。

## 5. 产品构建身份

一个产品构建身份识别实际运行所需的源码、锁定依赖、迁移、前端制品和非秘密配置。研究、普通文档和未进入产品运行的工具不改变产品构建身份。

App、Executor、固定计划和真实账户交易开关使用同一个身份；不一致时拒绝相应交易所变更请求。该身份只用于追溯和输入一致性，不形成发布资格、第二摘要或长期观察流程。当前值和生成方式由代码与 L4 记录。

## 6. 秘密、备份与故障

秘密使用操作系统或等价外部秘密存储，不进入仓库、业务数据库、前端、日志或构建记录。执行角色只有在产品构建、数据库、唯一执行者、当前计划、资金检查和账户事实都满足后，才读取真实账户交易秘密。

数据库变更串行迁移；不可逆变化前有可用备份和还原检查。备份、恢复或回退失败时，不允许发起依赖它们的真实账户交易动作。

当前 Windows 版本、进程管理方式、凭据位置、命令和实际验证由 L4 记录，不固化为长期平台资格。
