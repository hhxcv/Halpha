# Halpha 交易事实原则

**文档编号：** HALPHA-DAT-001  
**层级：** L2  
**语言版本：** zh-CN  
**上位文档：** HALPHA-CON-001、HALPHA-FLOW-001、HALPHA-ARC-001  
**本文档负责：** 交易事实的来源、身份、时间、未知、修正和保留边界  
**本文档不负责：** 研究、计划、资金、执行或复盘决定，以及当前数据源和阈值

## 1. 唯一职责

DAT 让消费者知道一个事实属于什么环境和对象、来自哪里、截至何时、值是什么以及是否适合当前用途。供应商缓存、页面、日志和程序猜测都不能自行成为权威事实。

## 2. 来源、身份与时间

可被产品引用的事实必须保留足以识别环境、对象、来源、外部身份、值、单位、精度、来源时间、取得时间和事实截止点的信息。派生事实还必须引用输入和方法。

历史研究、回放、Demo 和 Live 的身份分离。价格或时间接近不能替代稳定身份；模拟和用户外部操作不能冒充 Halpha 的真实动作事实。

消费者按用途决定新鲜度和完整性。时钟异常、乱序、重复、缺口和未闭合周期必须可识别，不能静默补造或沿用最近值。

## 3. 未知与修正

来源缺失、过期、冲突、身份不明、单位或精度不明、序列不连续或推导输入不足时，受影响事实为未知。未知不是零、无订单、无持仓或安全状态。

消费者只停止直接依赖该事实的行为：新增风险拒绝；可靠查询、已有保护、退出和用户接管继续。后来取得的事实支持新的判断，不能改写当时可知范围。

外部修正或更可靠来源改变事实时，追加新版本并保留原事实及原因；DAT 不反向修改计划、动作或复盘。

## 4. 保留与复杂度

承担未闭合动作、持仓、保护、核对或复盘责任的事实必须保留。可从固定来源低成本重建的材料按当前需要清理。

当前一个事实记录结构能够表达时，不建立来源登记、质量任务、修正工作流、全量行情仓库、第二账本或第二事实数据库。详细字段和当前支持事实种类进入 L3/L4。
