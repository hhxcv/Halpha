# Halpha 带来源交易事实原则

**文档编号：** HALPHA-DAT-001  
**层级：** L2  
**语言版本：** zh-CN  
**上位文档：** HALPHA-CON-001、HALPHA-DOC-001、HALPHA-VIS-001、HALPHA-FLOW-001、HALPHA-ARC-001  
**本文档负责：** 交易事实的来源、环境、身份、时间截止、用途适用性、未知、修正和保留边界  
**本文档不负责：** 作出研究、计划、资金、执行或复盘决定；保存全量供应商数据；定义具体字段、来源、阈值、保留期、组件或当前配置  

---

# 0. 唯一职责与复杂度上限【DAT-SCP-001】

DAT 只负责让消费者知道“关于哪个对象、来自哪里、截至何时、值是什么、能否用于当前决定”。它不拥有交易决定，也不把供应商缓存、程序推断、页面显示或日志提升为权威事实。

外部观察、规范化值、派生值、未知和修正不需要分别建立产品对象。它们是同一事实记录的来源、形成方式、质量和版本属性。当前一个记录族能够表达时，不再建立来源登记、质量判断、追溯图、修正任务或第二事实仓库。

# 1. 带来源事实【DAT-FCT-001】

一条可被产品引用的事实必须能够识别：所属环境和对象；来源及外部身份；观察值、单位和精度；实际发生或来源标注时间；Halpha 取得时间；本事实覆盖到的截止点；以及必要时的输入与计算方法。

直接观察与派生值使用同一事实形状，但派生值必须引用输入和方法，不能伪装成外部观察。供应商对象、推送顺序、时间接近或相似数值不能替代稳定身份。消费者只能使用满足其对象、时效、完整性和失败要求的事实。

DAT 负责市场、工具规则、账户、余额、保证金、订单、成交、费用、资金费、仓位和保护等实际需要的事实；具体支持集合由 L3/L4 收缩，不要求为未来来源预建空类型。

# 2. 环境与身份隔离【DAT-ENV-001】

历史研究、历史回放、Demo 和 Live 必须能被明确区分。账户、工具、订单、成交、仓位、保护和动作归属只能在同一环境和稳定身份内关联；环境缺失或冲突时，不形成可支持交易的事实。

Demo 观察、动作和结果不得复制、改名或提升为 Live 事实。用户在场所官方入口完成的活动保留为外部活动；只有稳定外部身份和实际证据能够对应 Halpha 动作时，才建立归属关系。

# 3. 来源、时间与当前可用性【DAT-SRC-001】

来源是否可用由当前消费决定，而不是由来源名称、数量或历史成功决定。市场判断、资金检查、场所提交、保护和复盘可以使用不同新鲜度与完整性要求；具体阈值进入 L3/L4。

至少区分来源所标注的发生时间、Halpha 取得时间和本次事实截止点。其他处理时间只在当前消费者确实需要时保存；不得为了完整模板固定五套时间字段。时钟异常、乱序、重复、缺口和未闭合周期必须可识别，不能静默排序或补造。

切换来源前必须验证对象身份、单位、精度、时间、连续性和失败语义兼容。无法证明等价时，旧来源失败的结果是未知或明确不支持，不自动把第二来源当作同一事实。

# 4. 未知与消费者行为【DAT-UNK-001】

来源缺失、过期、冲突、身份不明、序列不连续、单位或精度不明、推导输入不足时，受影响事实保持未知。未知不是零、否、无持仓、无订单或安全状态。

每个消费者只停止直接依赖该事实的行为：研究和页面显示限制；计划不产生依赖它的动作；CAP 和 EXE 拒绝依赖它的新增风险；可靠只读查询、已有保护、退出和用户接管继续。后来取得的数据可以支持新决定，但不能改写当时可知范围。

# 5. 修正、保留与重建【DAT-COR-001】

外部修正、迟到结果或更可靠来源改变事实时，追加引用原事实的新版本，说明原因、来源、时间和受影响范围；原事实不得覆盖。消费者根据自身规则决定是否重算或追加自己的结果，DAT 不直接改写计划、动作或复盘。

事实及其来源关系至少保留到依赖它的动作、持仓、保护、核对和复盘责任闭合。能从固定来源低成本重建的数据可以按当前策略清理；不能重建且仍承担责任的事实必须保留并可导出。精确保留期、迁移和导出格式属于 L3/L4。

# 6. 下位设计与当前记录【DAT-L3-001】

L3 只定义当前消费者需要的事实种类、最小字段、稳定身份、来源映射、时间与新鲜度、追加修正、查询、恢复和关键反例。一个事实记录族足够时，不增加缓存、checkpoint、质量任务、追溯服务或第二数据库。

L4 只记录当前场所、来源、账户、工具、时间窗、阈值、保留期、组件版本和直接验证结果【DAT-L4-001】。缺少当前结果时保持未知，不从设计完整或历史成功推定当前可用。
