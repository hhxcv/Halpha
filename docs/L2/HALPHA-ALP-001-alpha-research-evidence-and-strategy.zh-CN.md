# 0. 唯一职责【ALP-SCP-001】

ALP 只负责回答一个经济主张或固定交易策略在明确适用范围、基准、成本、现实约束和不确定性下得到何种支持，以及哪些结果可以作为交易计划形成的依据。

ALP 不拥有研究或交易计划候选、用户的资金用途选择、交易计划、计划启用、资金使用上限与范围、Halpha 真实资金操作权限或拟执行动作。研究、回测、前向观察、真实结果、模型评分和单次盈利均不能单独形成 Alpha 结论，也不能直接授权计划或真实动作。

是否建设正式研究或策略能力、支持哪些对象和方法，由 L4 根据真实消费者记录；没有当前消费者时保持本文件的边界深度。

# 1. 进入与退出边界【ALP-OBJ-001】

## 1.1 进入要求【ALP-OBJ-001-REQ】

正式经济研究只从以下输入开始：

- CTX 的研究或交易计划候选去向决定交付的经济研究起始材料；
- 用户直接提出且已经说明经济机制、被评价对象、决策用途与可证伪问题的正式研究问题；
- 既有计划或复盘暴露出的、确实可能改变经济判断的问题。

不能说明经济机制、适用范围、决策用途或可证伪问题的材料，不进入正式经济研究；可以退回 CTX、等待补充或明确不支持。ALP 不在自身范围内继续筛选研究或交易计划候选。

## 1.2 退出边界

ALP 只向下游交付以下一种结果：

- 对被评价对象在明确范围内的经济证据判断及其主要限制；
- 一个固定交易策略可向 TRADEPLAN 提供的计划依据及其适用边界；
- 不支持、证据不足或无法判断的明确结果。

上述结果都不是资金用途选择、交易计划、计划启用、资金权限或动作命令。消费者必须保留适用范围、主要未知和不能推出的结论。

# 2. 最低研究与证据要求

## 2.1 候选优势进入研究要求【ALP-ADV-001-REQ】

进入正式研究的候选优势至少必须说明：

- 可证伪的经济、行为或市场结构机制；
- 适用与不适用的市场、资产、场所、资金规模和时间条件；
- 数据、成本、流动性、执行和维护的现实可行性；
- 现金、不交易、现有方法或低成本外部工具中的适用比较对象；
- 最强反解释、已知反证、关键未知和可观察失效条件。

主要依赖极低延迟、队列优先权、专有基础设施、机构资本规模，或只有忽略成本与现实执行才成立的主张，默认不适合 Halpha；只有出现可验证的新条件时才重新研究。

## 2.2 研究材料与经济证据判断分离【ALP-EVD-001-REQ】

研究方案、运行结果、回测报告、模型评分、AI 输出和真实交易材料只能作为判断输入。形成可供下游引用的经济证据判断时，必须同时说明：

- 被评价对象及适用范围；
- 事前确定的比较对象、成本和判定依据；
- 数据与决定时点是否正确，实际搜索和失败是否被保留；
- 现实资金规模、流动性、执行与维护约束；
- 支持、不支持、证据不足或无法判断的结论强度；
- 主要不确定性、失效条件和当前是否仍可引用。

任一关键项缺失、过期、冲突或无法重演时，不得形成正向结论。AI、被评价策略、模型或研究运行不能使自己的正向判断生效。

# 3. 交易策略与计划交接【ALP-STR-001】

## 3.1 正式策略边界【ALP-STR-001-REQ】

交易策略只表达固定输入和条件怎样形成候选计划依据，并说明经济机制、适用范围、事实未知时的不输出规则、成本与失效条件。它不是交易计划、计划启用、资金权限或真实动作命令。

策略的经济机制、输入语义、适用范围、成本、风险或决定规则发生实质变化时，旧经济判断不得自动覆盖变化后的内容。Halpha 必须重新评价，或停止该策略继续提供新的计划依据。

TRADEPLAN 负责把用户判断、经济证据判断或固定策略依据形成完整交易计划。ALP 不复制计划状态、资金检查、动作条件或执行责任。

---

# 4. 失败与失效结论【ALP-NEG-001】

## 4.1 保留和复用要求【ALP-NEG-001-REQ】

不支持、证据不足、时间语义错误、数据泄漏、选择偏差、不可重演、成本或资金规模越界、前向未复现和现实维护不可行等结论，必须在仍会影响后续判断的范围内保留。

重新研究相同主张时，必须说明出现了什么新信息以及为什么可能改变原结论。只更换名称、模型、参数或样本起点，不构成新的经济机制。

---

# 5. 明确输入与输出【ALP-IO-001】

| 方向 | 最低边界 |
|---|---|
| 接收 | 正式经济问题、可追溯事实与材料、明确决策用途、适用范围、主要反证和未知 |
| 输出 | 有边界的经济证据判断，或固定策略向 TRADEPLAN 提供的计划依据；也可以是不支持、证据不足或无法判断 |

下游不得把 ALP 输出解释为用户资金用途选择、交易计划、提高资金使用上限或扩大范围、计划启用、提高 Halpha 真实资金操作权限、工程质量结论或真实动作许可。

详细字段、研究方法、测试和实现契约只有在 L4 记录真实消费者并选择相应建设范围后才进入 L3；本文件不预建对象生命周期、成熟度状态体系、自动化流程或研究平台。
