# Halpha 复盘原则

**文档编号：** HALPHA-OUT-001  
**层级：** L2  
**语言版本：** zh-CN  
**上位文档：** HALPHA-CON-001、HALPHA-VIS-001、HALPHA-FLOW-001  
**本文档负责：** 一次计划运行的结果复盘和后续问题  
**本文档不负责：** 修改计划、资金、事实、动作和相邻责任，以及自动证明 Alpha

## 1. 唯一职责

OUT 用一份简短复盘连接一次计划运行的原判断、计划、资金边界、动作、场所结果、成本、未知和开放责任，帮助用户决定下次是否改变做法。

复盘不能关闭订单或持仓责任、恢复旧计划、扩大资金范围或修改源记录。盈利不能掩盖重复动作、保护缺口和未知；亏损也不能单独证明策略或系统错误。

## 2. 复盘内容

成交、未成交、拒绝、过期、部分结果、未知和用户接管都可以复盘。复盘只引用适合本次评价的来源和事实截止点，无法可靠归属的费用、盈亏和外部活动保持未知。

评价只保留会改变后续决定的内容：计划和条件是否按固定规则工作；资金边界是否生效；动作、保护和核对是否正确；用户控制或系统运行是否造成重要偏差。

后来事实改变评价时追加修正并保留原结论。单次结果、模拟盈利和复盘评价不能自动证明 Alpha 或提高资金范围。

## 3. 后续问题与复杂度

确有可能改变后续决定的问题，用普通文本记录问题、依据、影响和建议所有者。只有项目所有者决定处理时，才另行开始研究或开发；OUT 不自动创建任务、审批或改进流程。

一个简短复盘记录足够时，不建设评分、聚合、分析平台、独立 worker 或第二交接对象。详细结构进入 L3，当前启用范围和结果进入 L4。
