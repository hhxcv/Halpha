# 0. 唯一职责【CTX-SCP-001】

## 0.1 CTX 责任定义【CTX-SCP-001-DEF】

CTX 只负责把尚未由其他领域承担现实责任的具体问题、机会或风险线索识别为研究或交易计划候选，并支持用户决定它下一步进入正式经济研究、进入交易计划形成、等待后再决定、结束，或当前不形成交易计划。

## 0.2 CTX 责任边界要求【CTX-SCP-001-REQ】

CTX 不证明经济优势，不替用户作投资判断，不把来源材料或 AI 输出升级为权威事实，也不形成研究结论、交易计划、资金决定、权限或真实动作。来源数量、信息丰富度、优先级和 AI 置信度均不具有上述效力。

已经形成订单、持仓、保护、待执行动作、外部结果、事实冲突、停用或运行责任的事项，必须直接交给 EXE、DAT、CAP、SYS、ENG 或 UX；不得放入研究或交易计划候选等待处理。相应责任稳定后若产生新的研究或计划问题，才可以重新进入 CTX 边界。

# 1. 必要概念

## 1.1 研究或交易计划候选定义【CTX-CAN-001-DEF】

研究或交易计划候选，是一个尚未由其他领域承担处理责任、待决定是否进入正式经济研究或交易计划形成的具体问题、机会或风险线索。“候选”只表示去向尚未确定，不表示已经具有经济价值、交易资格或计划效力。

## 1.2 研究或交易计划候选去向决定定义【CTX-DEC-001-DEF】

研究或交易计划候选去向决定，是用户针对一个明确候选作出的下一步选择。只有在已接受规则明确且不需要用户承担主观交易判断时，Halpha 才可以形成同类选择。该决定不形成研究结论、交易计划或资金动作。

---

# 2. 输入与去向边界

## 2.1 输入边界【CTX-CAP-001-REQ】

输入可以来自用户、Halpha 线索、外部材料、通用 AI、既有计划或策略暴露的新问题，以及确实需要重新研究或形成新计划的复盘改进事项。输入至少应使用户能够识别问题、来源、时点、适用对象、主要未知和下一步选择；无法恢复原意时，只能等待补充或结束，不得支持涉及真实资金的正向判断。

外部材料、AI 回答和用户记忆默认只是带来源的主张或意图。权威事实、事实未知与修正由 DAT 拥有，CTX 只消费其对本次去向选择的影响。

## 2.2 去向选择要求【CTX-RTE-001-REQ】

一次研究或交易计划候选去向决定只能选择一个主要结果：

| 结果 | 交接边界 |
|---|---|
| 进入正式经济研究 | 向 ALP 提供研究问题、机制线索、适用边界、主要反证与未知；不输出经济证据判断 |
| 进入交易计划形成 | 向 TRADEPLAN 提供交易意图、对象、窗口、依据、替代选择与会改变决定的未知；不输出交易计划 |
| 等待后再决定 | 明确等待条件、最晚复查时间和条件未到达时的结束结果 |
| 结束该候选 | 说明重复、无关、不可判定、失去时效、成本不成比例或超出边界等理由 |
| 当前不形成交易计划 | 只结束当前候选的计划形成路径，不成为永久市场判断，也不改写既有交易计划 |

关键事实、问题边界或有效窗口不清时，只能等待、结束或交由用户决定；不得把未知解释为进入研究或计划的正向依据。已有交易计划后的不行动、过期、结束或停止由 TRADEPLAN 拥有，不由 CTX 重新分类。

---

# 3. 交付与缺失结果【CTX-IO-001】

CTX 只交付以下边界结果：

- 向用户和 UX 交付一个可理解的研究或交易计划候选及其当前去向；
- 向 ALP 交付正式经济研究所需的最小起始材料；
- 向 TRADEPLAN 交付形成具体计划所需的最小起始材料。

消费者不得把候选、推荐、优先级或去向决定解释为经济支持、计划接受、资金权限或真实动作许可。输入不足、事实未知、消费者未建设或交接失败时，结果必须是等待、结束、人工计划或明确不支持，不得为闭合流程新增对象、自动化或临时权限。

当前是否建设 CTX、支持哪些输入渠道和去向，由 L4 记录；没有当前消费者时保持本文件的边界深度。
