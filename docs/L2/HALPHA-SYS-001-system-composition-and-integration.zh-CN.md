# Halpha 系统组成与运行原则

**文档编号：** HALPHA-SYS-001  
**层级：** L2  
**语言版本：** zh-CN  
**上位文档：** HALPHA-CON-001、HALPHA-ARC-001  
**本文档负责：** 模块所有权、事务、运行角色、交易所变更请求隔离、启动停止和恢复组合  
**本文档不负责：** 业务对象含义、页面、当前进程数量、组件版本和部署配置

## 1. 唯一职责

SYS 只定义业务模块怎样组合、谁可以写什么、事务与外部动作怎样分界、运行角色怎样取得和失去能力，以及系统怎样启动、停止和恢复责任。业务含义仍由各领域拥有；工程验证和部署方法由 ENG 拥有。

## 2. 所有权与依赖

每项可变业务状态和决定只有一个拥有模块。其他模块只能读取已提交状态或请求拥有模块修改，不能直接写私有状态、复制状态机或从页面、缓存和日志反推权威结果。

同一进程内默认直接调用拥有模块。需要跨请求继续、能够独立失败且必须从持久状态恢复的工作，才使用持久交接；不建设通用命令、事件或任务平台。

## 3. 事务与外部动作

一次本地业务变化在明确事务中全部成立或全部不成立。外部动作不能与数据库伪装成一个事务；EXE 先持久动作责任，再由隔离写边界处理。

可重建视图和缓存只服务查询。外部适配只处理协议、身份、单位、精度、限频和错误，不取得业务决定权。

## 4. 运行角色与秘密

系统保持交互应用和隔离执行两类逻辑角色：交互角色处理用户请求和查询，不持有真实账户交易凭据；执行角色处理已持久化的动作责任，并且是唯一可以发起交易所变更请求的角色。

这些角色是否映射为不同进程、实例或账户由 L3/L4 决定。配置不能创造业务权限、恢复被停止的计划或跳过当前检查；秘密只注入确需它的运行角色。

## 5. 启动、停止与恢复

SYS 是系统恢复顺序的唯一所有者。启动或非计划恢复依次确认运行身份和唯一交易所请求执行者、持久状态与迁移、非秘密配置、外部当前事实和未闭合责任；只有各拥有领域返回一致且不会补发错过动作时，系统才继续原责任。

条件不足时停止受影响新增风险和新的交易所变更请求，保留查询、已有保护、退出和用户接管。各领域只返回自己状态的恢复结果，不在本领域文档复制完整系统顺序。

正常停止先阻断新的交易所变更请求，再保存和交接未闭合责任。备份和还原恢复权威数据，不恢复已经失效的真实账户交易权限、会话或过期决定。

## 6. 演进边界

模块化单体是默认。只有实际并发、故障隔离、独立启停或维护问题证明收益高于复杂度时，才增加进程、存储、消息设施或持久工作；当前数量和映射只记录在 L4。
