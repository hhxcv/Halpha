# Halpha 核心业务流程与用户路径

**文档编号：** HALPHA-FLOW-001  
**层级：** L1-C  
**语言版本：** zh-CN  
**上位文档：** HALPHA-CON-001、HALPHA-DOC-001、HALPHA-VIS-001  
**本文档负责：** 长期用户路径、人与系统分工、外部工具交接和产品级失败结果  
**本文档不负责：** 页面、字段、内部状态、技术事务和当前建设

## 1. 核心路径

Halpha 的主线是：看清事实，形成判断，制定计划，决定是否启用，观察和执行，核对结果，停止或结束，并从结果中改进。

路径不强迫交易。等待、保持现金、不采用研究结果、计划过期、拒绝动作、停止和外部接管都是正常结果。

## 2. 进入与当前任务

用户进入产品时，应先看见与当前决定有关的账户、持仓、订单、运行计划、重要异常和未知。没有待处理事项时可以退出，或直接进入研究、计划和复盘。

中断后继续时，产品先显示期间变化、过期和冲突。已经发生或可能发生的外部动作先核对，不能因刷新、重启或重复点击重放。

## 3. 研究路径

研究从一个可说明的问题、数据边界和可证伪方向开始。AI 可以在授权的研究环境中探索、实现、失败、重跑和寻找反例；用户决定是否采用结果。

研究不需要交易产品运行，也不得因此访问产品数据库、真实凭据或场所写能力。准备把结果用于产品或资金决定时，再补足与结论强度相称的成本、反证、适用范围和可重演材料。

## 4. 计划与启用

用户可以从自己的判断或已采用的研究结果形成计划。计划说明为什么行动、何时行动和失效、承担多少、怎样保护和退出，以及事实未知或路径失败时怎样处理。

用户明确启用一个完整计划后，Halpha 才可以在该决定范围内持续观察并产生动作。启用不是立即成交，也不替代每次动作所需的当前事实和检查。修改会改变资金后果或行动条件时，用户形成新的计划决定。

## 5. 执行与核对

条件成立只产生计划内动作意图。实际外部动作必须使用当前事实，经过资金边界检查，由唯一执行路径处理并核对交易场所结果。重复、超时和结果未知不能产生猜测性的第二动作。

模拟和真实资金使用相同核心业务语义，但身份、账户、凭据、动作和结果分离。模拟结果不能成为真实资金事实或许可。

## 6. 停止、退出和接管

用户可以停止系统新增风险、要求按计划退出，或转到交易场所官方入口接管。停止不自动表示已经撤单或平仓；已有订单、持仓、保护和未知结果继续核对或明确交给用户。

Halpha 不可访问时，用户可以通过官方入口观察账户、处置风险和撤销凭据。产品恢复后先读取外部事实，不把用户外部操作改写为自身动作。

## 7. 结果与学习

计划没有行动、被拒绝、部分执行、完成、未知或被用户接管，都可以形成复盘。复盘连接原判断、计划、实际动作、成本、结果和未知，只形成可能改变后续决定的问题，不自动修改策略、计划或资金范围。

## 8. 领域交接

研究领域形成可采用的判断；交易计划领域拥有计划和启用生命周期；资金领域提供边界检查；事实领域提供带来源的当前事实；执行领域拥有外部动作责任；复盘领域形成评价；交互、系统和工程领域提供跨业务约束。

这些责任不要求同名模块、页面或相同投入。具体对象和失败语义由各自 L2/L3 定义，当前支持范围只由 L4 记录。
