# 1. 产品身份与长期使用背景【VIS-IDN-001】

## 1.1 继承的长期产品定位

本文不重新定义 Halpha、用户、项目所有者或开发者；在 `CON-MIS-001` 与 `CON-USR-001` 的边界内，长期产品定位为：

> Halpha 是用户投入交易资本并使用的个人交易决策、执行与学习系统。它以更高质量的市场判断、策略、计划、便捷交互、可靠执行和持续学习改善用户的真实交易结果；项目所有者以相称的开发与维护成本评价产品成功。

## 1.2 长期设计依据【VIS-IDN-001-RAT】

长期产品定位基于以下使用背景：

- 用户投入自有交易资本并承担交易结果，同时采用主观判断、事件分析和量化研究；
- 交易资本、建设资源和长期维护负担都属于个人尺度，不以机构团队、基础设施或组织流程为前提；
- 市场竞争激烈且持续变化，显而易见的机会会被快速利用，历史有效性也会衰减；
- 用户没有专职研究、执行、风险和运营团队，可用时间有限且经常碎片化；在信心较低时会谨慎观察和频繁干预，在结果稳定且边界清楚时可以减少关注；
- 用户理解基本交易概念，愿意承担必要学习成本，并期望获得专业但不过度繁琐的分析、计划和控制能力；
- 用户愿意事前表达经济假设、主要反证、失效条件、风险边界、计划条件和评价方法，并会同时使用交易所、专业图表、新闻、数据、研究环境和通用 AI 等工具。

## 1.3 产品身份保持要求【VIS-IDN-001-REQ】

产品必须以个人使用模式为设计基准，不要求用户编程、维护数据管道、处理内部状态机或阅读系统日志。复杂度必须保持在用户和项目所有者能够理解、验证、运行、接管、恢复和退出的范围内；产品始终服务单一用户投入的交易资本，不以多用户、多机构、客户资金流程或无人值守替代用户判断为目标。

产品身份不随当前实施范围、进度或技术选择变化。实际市场、场所、账户、标的、周期和当前支持范围由 L4 记录。符合第 5.3 节竞争边界且不构成 `CON-NGL-001` 所称扩张的能力调整，不要求改变产品身份；改变资金责任主体、用户模型、运营模式、交易时间尺度、商业性质或竞争结构的扩张，必须先按宪法复核。

# 2. 要解决的真实问题【VIS-PRB-001】

## 2.1 问题与判断依据【VIS-PRB-001-RAT】

### 信息很多，但不能直接形成资金用途或真实动作判断

行情、新闻、链上信息、研究观点和模型结果持续增加。困难不在于取得更多内容，而在于识别哪些变化与当前账户、持仓、资金使用、假设和候选优势有关，同时保留反对证据、未知和失效条件。

### 人不能持续研究和值守

有价值的条件可能在用户离线期间出现，候选优势也需要反复更新和证伪。要求用户持续浏览、研究和盯盘，会把系统缺失转化为人的长期值班负担。

### 好判断可能被坏计划和坏执行破坏

临时改变论点、追价、重复下单、忽略成本、部分成交后未保护，或触发后不重新检查市场与账户，都可能吞噬原本有限的优势。

### 真实结果没有自然变成竞争证据

成交、未触发、过期、拒绝、人工干预、保护和不交易往往散落在不同工具中。没有事前假设、评价方法、真实成本、真实动作证据和账户核对，就无法区分判断、信号、执行、Beta、杠杆、运气和 Halpha 的贡献。

### AI 同时放大研究效率和伪优势

AI 可以快速生成解释、策略和代码，也可以更快制造数据泄漏、过拟合、选择性叙事和难以验证的复杂度。多个同源 AI 结论不构成独立证据，AI 产量也不是竞争力。

### 表面盈利很容易被误认为 Alpha

市场 Beta、杠杆、一般风险溢价、幸存偏差、事后改口径、少数极端交易、未计费用和运气都可能产生漂亮结果。

### 优势微弱、有条件，并且会衰减

真实优势通常只适用于特定市场状态、资产、周期、资金规模、流动性和成本环境，并可能因参与者拥挤、市场适应、数据变化或执行成本上升而消失。找到一个策略不是终局，退化和替换是正常产品问题。

### 研究与真实资本之间存在断层

研究时的数据可得性、成交假设、流动性和费用与真实运行不同，产品必须面对这项差异而不能用模拟结果代替真实结果。

## 2.2 由问题产生的产品要求【VIS-PRB-001-REQ】

- 研究、计划、用户决定、执行和结果必须能够对应，但具体对应方式由 FLOW 和下位设计定义。
- `CON-CMP-003` 定义的金融风险控制、功能正确性和系统自身风险缓解必须分别评价，任何一类结果都不能替代另外两类。
- Halpha 必须在事前基准、成本、风险和适用范围下辨认真正的超额投资结果。
- 无法在时点正确数据、样本外或前向环境以及真实执行约束中保持的优势，不能成为资本竞争力。

# 3. 产品价值来源【VIS-VAL-001】

## 3.1 产品价值类别定义【VIS-VAL-001-DEF】

Halpha 的长期产品价值分为以下八类；这些类别描述产品要改善的结果，不表示模块、运行实体、实施单元或先后顺序：

| 价值来源 | 产品要带来的改变 |
|---|---|
| 交易与盈利价值 | 形成更有竞争力、可证伪且有适用范围的观点与策略，改善机会选择、进入退出和资本使用结果 |
| 判断与计划价值 | 让事实、假设、反证、未知和失效条件更清楚，并把判断变成可执行、可调整的计划 |
| 执行价值 | 让用户决定或已授权计划在明确边界内得到唯一、可核对的落实，减少遗漏、延迟、歧义、重复、滑点和非计划干预 |
| UX 价值 | 让用户用少量步骤看清状态、比较方案、完成行动、停止系统和人工接管，减少误操作与注意力浪费 |
| 稳定性价值 | 让核心能力以成熟技术和简单结构在预期条件下可靠运行，而不以平台或基础设施数量证明价值 |
| 系统自身风险缓解价值 | 在主机、依赖、网络、凭据或运行环境失效时限制影响，并保持停止、恢复和外部接管可用 |
| 学习价值 | 从真实计划、成本和结果中判断优势是否仍有证据支持，淘汰错误做法并改进下一次判断 |
| 金融风险控制价值 | 在系统正确运行时仍执行用户配置的资金使用上限与范围和必要动作前检查，避免 Halpha 越过允许范围 |

“执行”“稳定性”“系统自身风险缓解”“金融风险控制”是不同价值类别。真实动作唯一、防重复、状态正确和结果核对属于基本功能正确性，不因当前实现范围或资金规模而弱化；各类别的当前支持范围和实际投入只由 L4 记录。

## 3.2 产品价值要求【VIS-VAL-001-REQ】

Halpha 必须以真实账户的长期净资本价值为目标，并在存在与结论强度相称的可比材料时分别评价投资 Alpha 和产品增量价值；材料不足时保留无法判定。产品投入顺序必须服从 `CON-PRI-001` 和 `CON-CMP-001`，不得让辅助控制、治理或平台建设挤压交易判断、计划、执行、UX 和可靠运行。

市场盈利不是系统上线、接入 AI 或规范执行后的自然结果。长期盈利要求 Halpha 在成本与风险之后持续形成相对市场基准和可用替代方案的竞争优势，并能在优势衰减时及时证伪、降权和替换。

计划、执行、UX 和稳定性可以在投资 Alpha 尚未成立时形成产品增量价值；但若长期没有足以支持限定适用范围投资 Alpha 结论的经济证据，Halpha 的核心盈利主张就尚未成立。账户净结果、投资 Alpha 和产品增量价值沿用 `CON-ECO-002` 与 `CON-ECO-003`，不得把市场上涨、杠杆或短期盈利自动归因于 Halpha；等待、现金和不交易仍可以是正确资本结果。

# 4. 产品运行主张【VIS-OPS-001】

## 4.1 产品参与者边界

### 参与者边界【VIS-OPS-001-DEF】

真人角色及其资本、建设和开发责任完全继承 `CON-USR-001` 与 `CON-USR-002`，VIS 不另行定义。产品层只固定以下关系：用户作出产品使用和资本控制决定；项目所有者决定项目建设并承担维护成本；Halpha 在当前适用规则和用户决定内提供持续分析、计划、观察、执行和学习能力；外部系统与工具提供事实、计算、研究或场所能力。运行实体、进程和模块边界由 ARC、SYS 与下位设计拥有。

### 参与者要求【VIS-OPS-001-REQ】

开发活动、外部工具或 AI 输出不得被当作用户决定、账户事实或超出既有边界的行动权限。Halpha 必须持续提高策略、分析、计划、执行、稳定性和 UX 质量；“用户承担交易结果”或“项目所有者承担建设责任”不能成为忽视明显业务错误、功能缺陷或糟糕体验的理由。

## 4.2 用户交易计划的半自动执行是长期有效形态

### 半自动执行形态的选择依据【VIS-OPS-001-RAT】

用户交易计划可以来自用户原创判断，也可以吸收 Halpha 生成或改进的策略和分析。用户配置并明确激活有界计划后，由机器值守、触发、检查和执行，已经能够构成完整产品。对高不确定或证据不足的机会，用户可以不激活、缩小计划范围或继续研究；Halpha 不为此增加逐动作授权路径。具体时限和动作条件由 FLOW、CAP、TRADEPLAN 与 EXE 定义。

## 4.3 Halpha 真实动作必须服从用户配置的资金使用上限、范围与权限

Halpha 不得因策略更强、盈利更高、运行更自动或使用时间更长而自行提高资金使用上限、扩大范围、提高真实资金操作权限或恢复已停止的能力。用户系统外总投入不由 Halpha 判断；系统内资金使用上限、范围与真实资金操作权限沿用 L0 和 CAP。

Halpha 的长期产品形态使用一次明确计划激活来启用有界计划内行动，用户仍可随时停止、退出或通过交易场所官方入口接管；保护、撤销和风险减少等收缩动作不得被扩大解释为新增风险能力。具体动作条件、检查和核对链只由 FLOW 及相应 L2/L3 拥有，VIS 不重复规定。

## 4.4 多工具环境中的产品边界

交易所、专业图表、新闻、数据、研究环境和通用 AI 可以提供成熟的观察、计算、场所操作、研究与解释能力。Halpha 的产品责任是在这些工具之间保持判断、计划、账户含义、行动来源、结果与证据的连续关系，并让用户能够看清、查询、停止和接管；汇集展示不转移用户决定权、外部账户事实权威或下位领域的语义所有权。

外部工具能够独立完成任务且不会破坏高价值上下文时优先复用。只有当外部工具迫使用户反复重建交易决定所需上下文，或不能闭合计划、行动、核对与学习的连续关系时，Halpha 才补足必要能力。具体工具交接由 FLOW 定义，技术取舍由 ARC 和下位设计定义。

## 4.5 专业深度不等于日常复杂度

Halpha 在所有交互方式中只维护一套交易与资金语义，并保留与用户使用场景相称的高级分析和有界定制能力。专业性来自事实、证据、风险、权限、未知和结果的准确表达，不来自信息密度、配置数量、技术术语或持续占用注意力。

Halpha 应通过上下文继承、合理默认、模板和自动化吸收内部复杂度；正常使用不要求用户操作内部机制。简化不得隐藏重要风险、反证、未知、资金使用上限与范围、真实资金操作权限、系统能力范围和真实结果，也不得替用户作最终交易判断或决定系统外个人资金安排。具体信息层次和交互方式由 FLOW 与下位设计确定。

# 5. 产品差异化、竞争力与 Alpha 构建【VIS-ADV-001】

## 5.1 产品层派生术语与应用

### 候选优势定义【VIS-ADV-001-DEF】

账户净结果、投资 Alpha 和产品增量价值完全采用 `CON-ECO-002` 与 `CON-ECO-003` 的含义，VIS 不另行定义。

**候选优势**是具有可证伪经济机制、适用范围、成本假设和失效条件的产品级假设，不等于投资 Alpha。支持投资 Alpha 结论的经济证据仍按 L0 和 ALP 定义的基准、成本、风险调整、评价方法、结论强度、适用范围和有效状态判断，不另设产品层证据名称。

### 派生术语使用要求【VIS-ADV-001-REQ】

候选优势不是永久属性或收益保证；回测结果、一次盈利或产品叙事不能自行提高经济证据结论。

## 5.2 个人量化方向的选择依据【VIS-ADV-001-RAT】

小资金、单一用户和 AI 并不构成全面优势。相对专业机构，Halpha 通常缺少数据与接入条件、基础设施、专业分工、运行冗余、资金与交易成本优势；通用 AI 也对其他参与者开放，本身不是壁垒。

可能形成非对称性的条件是：个人资本不需要容纳机构规模，可以只选择少量机会并长期保持现金；单一用户协调成本较低；用户专属知识、判断与真实结果可以长期积累；AI 能降低研究、开发、比较和证伪的固定成本；系统能够持续观察和重复执行，使有限且碎片化的人类注意力集中于关键判断。

这些只是值得检验的资源条件，不是既有 Alpha。AI 可以放大研究与运营能力，但不能消除信息、资本、基础设施和现实执行约束；产品过程更严谨也不能自动证明投资 Alpha。

## 5.3 竞争范围要求【VIS-ADV-001-REQ-002】

### 有现实机会的竞争结构

Halpha 优先研究符合以下结构的机会：

- 机会只适用于较小资金规模，或对个人资本有意义，却难以容纳机构资本或覆盖机构固定成本；
- 选择性本身有价值，用户可以聚焦窄范围、等待或不交易，而不必持续部署交易资本；
- 候选优势可能来自用户专属上下文、跨来源综合、快速证伪和真实结果积累，而不是可廉价复制的单一公共信号；
- AI 能降低个人研究和开发的固定成本，并把输出沉淀到专属上下文和证据闭环；
- 候选优势在现实信息可得性、决策与执行约束、成本、流动性和风险之后仍可能成立；
- 研究、运行、维护、接管和退出能够由单一用户与项目所有者长期承担。

Halpha 应在这些条件内扩大可验证的能力范围，不因当前能力有限而永久排除某个持仓周期、交易频率或策略家族。具体机会仍必须按事前口径单独证伪和评价，不能由“个人项目”身份或对机构行为的推测直接推出。

### 结构性不利的竞争方向

以下竞争结构对 Halpha 几乎没有现实机会，不作为产品能力方向：

- 胜负主要由更快响应、更优优先级、更大资本、更低资金成本或交易成本决定；
- 依赖个人项目无法合理取得或持续维护的专有数据、接入条件、基础设施、连续运营、运行冗余或同步协调能力；
- 使用公共信息和通用 AI 即可廉价复制，而 Halpha 没有额外的用户专属上下文、证据积累或反馈优势；
- 只有忽略现实成本、流动性、执行不确定性、临时风险敞口或已配置资金使用上限与范围时才显得成立；
- 为获得足够经济价值，必须把覆盖范围、动作负担或维护复杂度扩大到机构级别。

持仓长短、交易次数和策略标签都不是独立判据。任何机会都不得以自动提高资金使用上限、扩大范围、削弱可核对性或失去用户最终控制换取参与机会。L1 只确定稳定竞争边界，不批准或永久排除具体策略；策略原则、详细契约和当前支持范围分别由适用的 L2、L3 和 L4 拥有。

# 6. 产品长期能力范围【VIS-CAP-001】

## 6.1 能力范围

以下是产品必须能够按实际价值逐步组合的长期能力，不表示软件模块、运行实体或同等建设投入：

1. **市场观察与实时情报：** 识别与用户当前关注和资本使用有关的市场变化、机会与风险，而不把信息流变成交易刺激。
2. **策略研究与验证：** 组织假设、机制、替代解释、反证和现实约束，形成可证伪策略并淘汰无效假设。
3. **交易计划与演练：** 把判断转为完整、可调整、可停止并可在行动前检查的计划。
4. **交易执行与持仓管理：** 可靠处理计划内行动、保护、调整、退出和异常责任，并保持结果可核对。
5. **账户与交易事实：** 让用户看清当前与历史账户结果、外部活动、事实未知和未决责任，并保持模拟与真实环境可辨。
6. **回顾分析与学习：** 连接假设、计划、行动、成本和结果，区分策略、计划、执行、交互与外部因素的贡献。
7. **用户 UX 与待处理事项：** 为不同时间条件下的查看、比较、决定、停止、接管和深度工作提供清楚入口。
8. **金融风险控制与资金操作权限：** 执行用户已配置的资金使用上限与范围、真实资金操作权限和相称的动作前检查，不替用户决定系统外总投入。
9. **系统自身风险缓解：** 以简单隔离、停止、恢复和外部接管限制系统故障或失控的影响。

## 6.2 能力实现要求【VIS-CAP-001-REQ】

上述能力不要求同时具备，也不要求分别建设成模块。当前消费者所需的能力必须形成可用闭环；其余能力可以保持边界、使用外部工具、人工处理或明确不支持。

适度稳定性是共同质量约束，不是独立用户工作域或高投入产品线。优先选择成熟组件、少依赖和简单部署，并由核心数据与执行语义保证真实动作正确。AI 可以生成假设、研究实现和反证，但不能成为经济证据，不能改变评价口径、删除失败或依据自身结果为自身提高结论强度。

# 7. 必须连接的产品价值依赖【VIS-LOOP-001】

Halpha 的产品价值依赖以下关系同时成立：

~~~text
可信事实与可证伪判断
↔ 可执行计划与清楚的人类选择
↔ 有界、可靠、可停止的行动或不行动
↔ 可核对结果与真实成本
↔ 归因、学习、淘汰与替换
~~~

这些关系只表达价值互相依赖，不规定处理顺序、参与者交接、授权路径、对象状态、工具或页面；实际路径由 FLOW 拥有，详细语义由相应 L2/L3 拥有。任一实际路径都必须允许等待、拒绝、现金、不交易、停止或人工接管成为合理结果，而不能为了闭合图形强迫行动。

# 8. 用户应获得的结果【VIS-OUT-001】

Halpha 应使用户：

- 看清与交易决定有关的账户、资金、订单、仓位、保护、差异和未知；
- 用包含论点、反证、失效条件、风险和资金使用上限与范围的计划行动；
- 用少量步骤完成高频分析、决定、查看、停止和人工接管；
- 在谨慎观察或减少关注时，都能获得有优先级的提醒和清楚状态；
- 获得可靠运行、故障后可理解恢复和不重复真实动作的核心体验；
- 解释真实成本、结果和产品贡献，并诚实淘汰无效候选优势与策略；
- 在证据支持时形成限定适用范围的 Alpha，并能降权、暂停和替换衰减优势。

前六项可以在尚无投资 Alpha 时产生产品增量价值；任何结果都不自动提高资金使用上限、扩大范围或提高 Halpha 真实资金操作权限。

# 9. 长期能力协同【VIS-DEP-001】

## 9.1 能力协同要求【VIS-DEP-001-REQ】

核心交易与 UX 闭环可以和 Alpha 研究与前向证据并行建设，两类真实使用可以相互暴露问题。研究不得把复杂基础设施当作前提；执行和 UX 必须由真实业务场景验证；Halpha 自动发起以交易盈利为目的并引入、增加或转换风险的真实动作，必须晚于完整计划、功能正确性、用户资本控制和与该动作相称的经济证据。保护、撤销或降低既有风险等收缩动作仍按 FLOW 规定的允许来源和统一动作链处理，不因本条被改写为增险动作。适度稳定性随核心链取得，不另建平行稳定性产品线。

# 10. 长期非目标与复杂度约束【VIS-NGL-001】

## 10.1 长期非目标

本文件完整继承 `CON-NGL-001` 的项目级非目标，不在此重复定义。除此之外，以下产品形态也不是 Halpha 的长期目标：

- 交易入门教育、喊单、保证收益、社交跟单或代替用户承担账户、持仓和资金使用的最终责任；
- 面向要求用户全天多屏连续操作的全职交易工作台；
- 综合新闻聚合、通用专业图表替代品或以内容消费为中心的信息终端；
- 替代真实交易场所的通用多场所交易终端、机器人或策略市场；
- 允许任意脚本、临时模型、外部告警或 AI 输出直接连接真实资本的自动化平台；
- 机构级风险引擎、多级生产准入、重型事故流程、企业安全平台或需要专职团队维护的治理体系；
- 用交易数、策略数、回测数、提醒数、代码量或功能数量证明成功。

## 10.2 长期复杂度约束

- Halpha 只在明确且受限的市场、账户、策略和数据范围内闭合真实价值；Halpha 不决定用户系统外总投入。
- 专业横向能力优先复用成熟工具；Halpha 只补足保持判断、计划、行动、核对和证据连续性所需的产品能力。
- 账户事实、真实成本和核对不能延后；候选优势必须允许证伪、降权和退出。
- L1 的全面性只描述长期价值闭环和防止责任遗漏，不承诺各领域同等自动化或投入。当前支持范围和实际投入由 L4 记录，横向业务责任与纵向约束不要求分别成为模块。
- 复杂度和维护注意力首先分配给 ALP、TRADEPLAN、DAT、EXE 与 UX。CAP、SYS、ENG、安全、治理、文档及其他辅助能力的单项和合计负担，不得超过其对核心业务的实际收益，也不得形成比核心交易链更厚的体系。
- 领域可以通过缩小支持范围、减少变体与通用化、采用人工或外部实现来控制复杂度；但已支持路径的功能正确性、停止、核对和用户最终控制不能因此弱化。
- 理想化闭环不得成为预建机构级流程、平台和基础设施的理由；安全主张若长期挤压交易能力、UX 和可靠运行，本身就是方向偏离。

# 11. 方向失败信号【VIS-FAL-001】

出现以下持续现象时，应复核方向或收缩主张：

- 结果主要由 Beta、杠杆、偶然交易或未计成本解释，前向证据持续消失；
- 失败、搜索范围、不利结果和真实执行差异没有被保留和解释；
- 候选优势缺少机制、适用状态、资金规模与流动性边界、成本或失效条件；
- 交易判断、计划或策略长期错误频发，无法形成任何盈利竞争力；
- 核心执行、数据、UX 或稳定性问题持续造成错误动作、错失机会或高人工负担；
- Halpha 重复建设成熟横向工具，却没有形成计划、行动、核对和证据连续性；
- CAP、ENG、安全、治理或事故流程比核心业务更复杂，并持续拖慢价值交付；
- 日常维护和注意力成本长期高于判断、执行、UX、稳定性和学习价值；
- 长期只能证明记录与纪律价值，不能形成任何限定适用范围的 Alpha 证据。

最后一项表示 Alpha 主张尚未成立，不否定已经形成的其他产品价值。

# 12. 下位设计交接【VIS-HOF-001】

HALPHA-FLOW-001 必须把本文件的价值依赖落实为可反复使用的产品级路径，覆盖行动、不行动或等待、事实未知、失败、停止、人工接管、恢复和结束，并明确 Halpha、用户与外部工具的交接；FLOW 不得把第 7 节当作固定步骤或对象状态机。

HALPHA-ARC-001 必须优先支撑可替换的 Alpha 与竞争力能力、顺畅 UX、可靠数据与执行，并通过成熟组件、简单拓扑和少依赖获得个人可维护的稳定运行。真实动作路径必须分别落实金融风险控制、功能正确性和系统自身风险缓解；实际支持范围由完整链路结果证明，不能由策略名称、持仓时长、文档完成或当前实施标签推断。
